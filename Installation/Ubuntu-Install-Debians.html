Set locale [待校准@110]

Make sure you have a locale which supports UTF-8. If you are in a minimal environment (such as a docker container), the locale may be something minimal like POSIX. We test with the following settings. However, it should be fine if you're using a different UTF-8 supported locale. [待校准@177]

locale  # check for UTF-8

sudo apt update && sudo apt install locales
sudo locale-gen en_US en_US.UTF-8
sudo update-locale LC_ALL=en_US.UTF-8 LANG=en_US.UTF-8
export LANG=en_US.UTF-8

locale  # verify settings

Setup Sources [待校准@475]

You will need to add the ROS 2 apt repository to your system. [待校准@179]

First ensure that the Ubuntu Universe repository is enabled. [待校准@180]

sudo apt install software-properties-common
sudo add-apt-repository universe

Now add the ROS 2 GPG key with apt. [待校准@181]

sudo apt update && sudo apt install curl -y
sudo curl -sSL https://raw.githubusercontent.com/ros/rosdistro/master/ros.key -o /usr/share/keyrings/ros-archive-keyring.gpg

Then add the repository to your sources list. [待校准@182]

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/ros-archive-keyring.gpg] http://packages.ros.org/ros2/ubuntu $(. /etc/os-release && echo $UBUNTU_CODENAME) main" | sudo tee /etc/apt/sources.list.d/ros2.list > /dev/null

Install ROS 2 packages [待校准@479]

Update your apt repository caches after setting up the repositories. [待校准@516]

sudo apt update

ROS 2 packages are built on frequently updated Ubuntu systems. It is always recommended that you ensure your system is up to date before installing new packages. [待校准@187]

sudo apt upgrade

警告

Due to early updates in Ubuntu 22.04 it is important that systemd and udev-related packages are updated before installing ROS 2. The installation of ROS 2's dependencies on a freshly installed system without upgrading can trigger the removal of critical system packages. [待校准@517]

Please refer to ros2/ros2#1272 and Launchpad #1974196 for more information. [待校准@518]

Desktop Install (Recommended): ROS, RViz, demos, tutorials. [待校准@480]

sudo apt install ros-humble-desktop

ROS-Base Install (Bare Bones): Communication libraries, message packages, command line tools. No GUI tools. [待校准@481]

sudo apt install ros-humble-ros-base

Development tools: Compilers and other tools to build ROS packages [待校准@519]

sudo apt install ros-dev-tools

# Replace ".bash" with your shell if you're not using bash
# Possible values are: setup.bash, setup.sh, setup.zsh
source /opt/ros/humble/setup.bash

Try some examples [待校准@129]

Talker-listener [待校准@520]

If you installed ros-humble-desktop above you can try some examples. [待校准@483]

In one terminal, source the setup file and then run a C++ talker: [待校准@130]

source /opt/ros/humble/setup.bash
ros2 run demo_nodes_cpp talker

In another terminal source the setup file and then run a Python listener: [待校准@131]

source /opt/ros/humble/setup.bash
ros2 run demo_nodes_py listener

You should see the talker saying that it's Publishing messages and the listener saying I heard those messages. This verifies both the C++ and Python APIs are working properly. Hooray! [待校准@132]

Using the ROS 1 bridge [待校准@190]

The ROS 1 bridge can connect topics from ROS 1 to ROS 2 and vice-versa. See the dedicated documentation on how to build and use the ROS 1 bridge. [待校准@191]

Additional RMW implementations (optional) [待校准@135]

The default middleware that ROS 2 uses is Fast DDS, but the middleware (RMW) can be replaced at runtime. See the guide on how to work with multiple RMWs. [待校准@484]

Uninstall [待校准@145]

If you need to uninstall ROS 2 or switch to a source-based install once you have already installed from binaries, run the following command: [待校准@486]

sudo apt remove ~nros-humble-* && sudo apt autoremove

You may also want to remove the repository: [待校准@521]

sudo rm /etc/apt/sources.list.d/ros2.list
sudo apt update
sudo apt autoremove
# Consider upgrading for packages previously shadowed.
sudo apt upgrade
