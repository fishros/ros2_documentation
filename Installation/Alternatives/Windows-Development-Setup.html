This guide is about how to setup a development environment for ROS 2 on Windows. [待校准@200]

System requirements [待校准@105]

Only Windows 10 is supported. [待校准@201]

Language support [待校准@202]

Make sure you have a locale which supports UTF-8. For example, for a Chinese-language Windows 10 installation, you may need to install an English language pack. [待校准@203]

Installing prerequisites [待校准@154]

Install Chocolatey [待校准@204]

Chocolatey is a package manager for Windows, install it by following their installation instructions: [待校准@205]

https://chocolatey.org/install [待校准@206]

You'll use Chocolatey to install some other developer tools. [待校准@207]

Install Python [待校准@208]

Open a Command Prompt and type the following to install Python via Chocolatey: [待校准@209]

choco install -y python --version 3.8.3

Install Visual C++ Redistributables [待校准@210]

Open a Command Prompt and type the following to install them via Chocolatey: [待校准@211]

choco install -y vcredist2013 vcredist140

Install OpenSSL [待校准@212]

Download the Win64 OpenSSL v1.1.1n OpenSSL installer from this page. Scroll to the bottom of the page and download Win64 OpenSSL v1.1.1t. Don't download the Win32 or Light versions, or the v3.X.Y installers. [待校准@213]

Run the installer with default parameters, as the following commands assume you used the default installation directory. [待校准@214]

This command sets an environment variable that persists over sessions: [待校准@215]

setx /m OPENSSL_CONF "C:\Program Files\OpenSSL-Win64\bin\openssl.cfg"

You will need to append the OpenSSL-Win64 bin folder to your PATH. You can do this by clicking the Windows icon, typing "Environment Variables", then clicking on "Edit the system environment variables". In the resulting dialog, click "Environment Variables", then click "Path" on the bottom pane, finally click "Edit" and add the path below. [待校准@216]

  • C:\Program Files\OpenSSL-Win64\bin\ [待校准@217]

Install Visual Studio [待校准@218]

Install Visual Studio 2019. [待校准@219]

If you already have a paid version of Visual Studio 2019 (Professional, Enterprise), skip this step. [待校准@220]

Microsoft provides a free of charge version of Visual Studio 2019, named Community, which can be used to build applications that use ROS 2. You can download the installer directly through this link. [待校准@221]

Make sure that the Visual C++ features are installed. [待校准@222]

An easy way to make sure they're installed is to select the Desktop development with C++ workflow during the install. [待校准@223]

Make sure that no C++ CMake tools are installed by unselecting them in the list of components to be installed. [待校准@224]

Install OpenCV [待校准@225]

Some of the examples require OpenCV to be installed. [待校准@226]

You can download a precompiled version of OpenCV 3.4.6 from https://github.com/ros2/ros2/releases/download/opencv-archives/opencv-3.4.6-vc16.VS2019.zip . [待校准@227]

Assuming you unpacked it to C:\opencv, type the following on a Command Prompt (requires Admin privileges): [待校准@228]

setx /m OpenCV_DIR C:\opencv

Since you are using a precompiled ROS version, we have to tell it where to find the OpenCV libraries. You have to extend the PATH variable to C:\opencv\x64\vc16\bin. [待校准@229]

Install dependencies [待校准@230]

There are a few dependencies not available in the Chocolatey package database. In order to ease the manual installation process, we provide the necessary Chocolatey packages. [待校准@231]

As some chocolatey packages rely on it, we start by installing CMake [待校准@232]

choco install -y cmake

You will need to append the CMake bin folder C:\Program Files\CMake\bin to your PATH. [待校准@233]

Please download these packages from this GitHub repository. [待校准@234]

  • asio.1.12.1.nupkg [待校准@235]

  • bullet.3.17.nupkg [待校准@236]

  • cunit.2.1.3.nupkg [待校准@237]

  • eigen-3.3.4.nupkg [待校准@238]

  • tinyxml-usestl.2.6.2.nupkg [待校准@239]

  • tinyxml2.6.0.0.nupkg [待校准@240]

Once these packages are downloaded, open an administrative shell and execute the following command: [待校准@241]

choco install -y -s <PATH\TO\DOWNLOADS\> asio cunit eigen tinyxml-usestl tinyxml2 bullet

Please replace <PATH\TO\DOWNLOADS> with the folder you downloaded the packages to. [待校准@242]

First upgrade pip and setuptools: [待校准@243]

python -m pip install -U pip setuptools==59.6.0

Now install some additional python dependencies: [待校准@244]

python -m pip install -U catkin_pkg cryptography empy importlib-metadata lark==1.1.1 lxml matplotlib netifaces numpy opencv-python PyQt5 pillow psutil pycairo pydot pyparsing==2.4.7 pyyaml rosdistro

Install Qt5 [待校准@245]

Download the 5.12.X offline installer from Qt's website. Run the installer. Make sure to select the MSVC 2017 64-bit component under the Qt -> Qt 5.12.12 tree. [待校准@246]

Finally, in an administrator cmd.exe window set these environment variables. The commands below assume you installed it to the default location of C:\Qt. [待校准@247]

setx /m Qt5_DIR C:\Qt\Qt5.12.12\5.12.12\msvc2017_64
setx /m QT_QPA_PLATFORM_PLUGIN_PATH C:\Qt\Qt5.12.12\5.12.12\msvc2017_64\plugins\platforms

注解

This path might change based on the installed MSVC version, the directory Qt was installed to, and the version of Qt installed. [待校准@248]

RQt dependencies [待校准@249]

To run rqt_graph you need to download and install Graphviz. The installer will ask if to add graphviz to PATH, choose to either add it to the current user or all users. [待校准@250]

If you would like to use another DDS or RTPS vendor besides the default, Fast DDS, you can find instructions here. [待校准@251]

Additional prerequisites [待校准@252]

When building from source you'll need a few additional prerequisites installed. [待校准@253]

Install additional prerequisites from Chocolatey [待校准@254]

choco install -y cppcheck curl git winflexbison3

You will need to append the Git cmd folder C:\Program Files\Git\cmd to the PATH (you can do this by clicking the Windows icon, typing "Environment Variables", then clicking on "Edit the system environment variables". In the resulting dialog, click "Environment Variables", the click "Path" on the bottom pane, then click "Edit" and add the path). [待校准@255]

Install Python prerequisites [待校准@256]

Installing additional Python dependencies: [待校准@257]

pip install -U colcon-common-extensions coverage flake8 flake8-blind-except flake8-builtins flake8-class-newline flake8-comprehensions flake8-deprecated flake8-docstrings flake8-import-order flake8-quotes mock mypy==0.931 pep8 pydocstyle pytest pytest-mock vcstool

Install miscellaneous prerequisites [待校准@258]

Next install xmllint: [待校准@259]

  • Download the 64 bit binary archives of libxml2 (and its dependencies iconv and zlib) from https://www.zlatkovic.com/projects/libxml/ [待校准@260]

  • Unpack all archives into e.g. C:\xmllint [待校准@261]

  • Add C:\xmllint\bin to the PATH. [待校准@262]

Get the ROS 2 code [待校准@263]

Now that we have the development tools we can get the ROS 2 source code. [待校准@264]

First setup a development folder, for example C:\humble: [待校准@265]

注解

It is very important that the chosen path is short, due to the short default Windows path limits (260 characters). To allow longer paths, see https://learn.microsoft.com/en-us/windows/win32/fileio/maximum-file-path-limitation?tabs=registry. [待校准@266]

md \humble\src
cd \humble

Get the ros2.repos file which defines the repositories to clone from: [待校准@267]

vcs import --input https://raw.githubusercontent.com/ros2/ros2/humble/ros2.repos src

Install additional DDS implementations (optional) [待校准@120]

Fast DDS is bundled with the ROS 2 source and will always be built unless you put an COLCON_IGNORE file in the src\eProsima folder. [待校准@268]

If you would like to use another DDS or RTPS vendor besides the default, you can find instructions here. [待校准@121]

Build the ROS 2 code [待校准@269]

To build ROS 2 you will need a Visual Studio Command Prompt ("x64 Native Tools Command Prompt for VS 2019") running as Administrator. [待校准@270]

To build the \humble folder tree: [待校准@271]

colcon build --merge-install

注解

We're using --merge-install here to avoid a PATH variable that is too long at the end of the build. If you're adapting these instructions to build a smaller workspace then you might be able to use the default behavior which is isolated install, i.e. where each package is installed to a different folder. [待校准@272]

注解

If you are doing a debug build use python_d path\to\colcon_executable colcon. See Extra stuff for debug mode for more info on running Python code in debug builds on Windows. [待校准@273]

Environment setup [待校准@126]

Start a command shell and source the ROS 2 setup file to set up the workspace: [待校准@274]

call C:\humble\install\local_setup.bat

This will automatically set up the environment for any DDS vendors that support was built for. [待校准@275]

It is normal that the previous command, if nothing else went wrong, outputs "The system cannot find the path specified." exactly once. [待校准@276]

Test and run [待校准@277]

Note that the first time you run any executable you will have to allow access to the network through a Windows Firewall popup. [待校准@278]

You can run the tests using this command: [待校准@279]

colcon test --merge-install

注解

--merge-install should only be used if it was also used in the build step. [待校准@280]

Afterwards you can get a summary of the tests using this command: [待校准@281]

colcon test-result

To run the examples, first open a clean new cmd.exe and set up the workspace by sourcing the local_setup.bat file. Then, run a C++ talker: [待校准@282]

call install\local_setup.bat
ros2 run demo_nodes_cpp talker

In a separate shell you can do the same, but instead run a Python listener: [待校准@283]

call install\local_setup.bat
ros2 run demo_nodes_py listener

You should see the talker saying that it's Publishing messages and the listener saying I heard those messages. This verifies both the C++ and Python APIs are working properly. Hooray! [待校准@132]

注解

It is not recommended to build in the same cmd prompt that you've sourced the local_setup.bat. [待校准@284]

Extra stuff for Debug mode [待校准@285]

If you want to be able to run all the tests in Debug mode, you'll need to install a few more things: [待校准@286]

  • To be able to extract the Python source tarball, you can use PeaZip: [待校准@287]

choco install -y peazip

  • You'll also need SVN, since some of the Python source-build dependencies are checked out via SVN: [待校准@288]

choco install -y svn hg

  • You'll need to quit and restart the command prompt after installing the above. [待校准@289]

  • Get and extract the Python 3.8.3 source from the tgz: [待校准@290]

    • https://www.python.org/ftp/python/3.8.3/Python-3.8.3.tgz [待校准@291]

    • To keep these instructions concise, please extract it to C:\dev\Python-3.8.3 [待校准@292]

  • Now, build the Python source in debug mode from a Visual Studio command prompt: [待校准@293]

cd C:\dev\Python-3.8.3\PCbuild
get_externals.bat
build.bat -p x64 -d

  • Finally, copy the build products into the Python38 installation directories, next to the Release-mode Python executable and DLL's: [待校准@294]

cd C:\dev\Python-3.8.3\PCbuild\amd64
copy python_d.exe C:\Python38 /Y
copy python38_d.dll C:\Python38 /Y
copy python3_d.dll C:\Python38 /Y
copy python38_d.lib C:\Python38\libs /Y
copy python3_d.lib C:\Python38\libs /Y
copy sqlite3_d.dll C:\Python38\DLLs /Y
for %I in (*_d.pyd) do copy %I C:\Python38\DLLs /Y

  • Now, from a fresh command prompt, make sure that python_d works: [待校准@295]

python_d -c "import _ctypes ; import coverage"

  • Once you have verified the operation of python_d, it is necessary to reinstall a few dependencies with the debug-enabled libraries: [待校准@296]

python_d -m pip install --force-reinstall https://github.com/ros2/ros2/releases/download/numpy-archives/numpy-1.18.4-cp38-cp38d-win_amd64.whl
python_d -m pip install --force-reinstall https://github.com/ros2/ros2/releases/download/lxml-archives/lxml-4.5.1-cp38-cp38d-win_amd64.whl

  • To verify the installation of these dependencies: [待校准@297]

python_d -c "from lxml import etree ; import numpy"

  • When you wish to return to building release binaries, it is necessary to uninstall the debug variants and use the release variants: [待校准@298]

python -m pip uninstall numpy lxml
python -m pip install numpy lxml

  • To create executables python scripts(.exe), python_d should be used to invoke colcon [待校准@299]

python_d path\to\colcon_executable build

  • Hooray, you're done! [待校准@300]

Uninstall [待校准@145]

  1. If you installed your workspace with colcon as instructed above, "uninstalling" could be just a matter of opening a new terminal and not sourcing the workspace's setup file. This way, your environment will behave as though there is no Humble install on your system. [待校准@146]

  2. If you're also trying to free up space, you can delete the entire workspace directory with: [待校准@147]

    rmdir /s /q \ros2_humble
