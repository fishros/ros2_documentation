This page explains how to install ROS 2 on Windows from a pre-built binary package. [待校准@7991]

注解

The pre-built binary does not include all ROS 2 packages. All packages in the ROS base variant are included, and only a subset of packages in the ROS desktop variant are included. The exact list of packages are described by the repositories listed in this ros2.repos file. [待校准@7618]

System requirements [待校准@7573]

Only Windows 10 is supported. [待校准@7669]

Installing prerequisites [待校准@7622]

Install Chocolatey [待校准@7672]

Chocolatey is a package manager for Windows, install it by following their installation instructions: [待校准@7673]

https://chocolatey.org/install [待校准@7674]

You'll use Chocolatey to install some other developer tools. [待校准@7675]

Install Python [待校准@7676]

Open a Command Prompt and type the following to install Python via Chocolatey: [待校准@7677]

choco install -y python --version 3.8.3

Install Visual C++ Redistributables [待校准@7678]

Open a Command Prompt and type the following to install them via Chocolatey: [待校准@7679]

choco install -y vcredist2013 vcredist140

Install OpenSSL [待校准@7680]

Download the Win64 OpenSSL v1.1.1n OpenSSL installer from this page. Scroll to the bottom of the page and download Win64 OpenSSL v1.1.1t. Don't download the Win32 or Light versions, or the v3.X.Y installers. [待校准@7681]

Run the installer with default parameters, as the following commands assume you used the default installation directory. [待校准@7682]

This command sets an environment variable that persists over sessions: [待校准@7683]

setx /m OPENSSL_CONF "C:\Program Files\OpenSSL-Win64\bin\openssl.cfg"

You will need to append the OpenSSL-Win64 bin folder to your PATH. You can do this by clicking the Windows icon, typing "Environment Variables", then clicking on "Edit the system environment variables". In the resulting dialog, click "Environment Variables", then click "Path" on the bottom pane, finally click "Edit" and add the path below. [待校准@7684]

  • C:\Program Files\OpenSSL-Win64\bin\ [待校准@7685]

Install Visual Studio [待校准@7686]

Install Visual Studio 2019. [待校准@7687]

If you already have a paid version of Visual Studio 2019 (Professional, Enterprise), skip this step. [待校准@7688]

Microsoft provides a free of charge version of Visual Studio 2019, named Community, which can be used to build applications that use ROS 2. You can download the installer directly through this link. [待校准@7689]

Make sure that the Visual C++ features are installed. [待校准@7690]

An easy way to make sure they're installed is to select the Desktop development with C++ workflow during the install. [待校准@7691]

Make sure that no C++ CMake tools are installed by unselecting them in the list of components to be installed. [待校准@7692]

Install OpenCV [待校准@7693]

Some of the examples require OpenCV to be installed. [待校准@7694]

You can download a precompiled version of OpenCV 3.4.6 from https://github.com/ros2/ros2/releases/download/opencv-archives/opencv-3.4.6-vc16.VS2019.zip . [待校准@7695]

Assuming you unpacked it to C:\opencv, type the following on a Command Prompt (requires Admin privileges): [待校准@7696]

setx /m OpenCV_DIR C:\opencv

Since you are using a precompiled ROS version, we have to tell it where to find the OpenCV libraries. You have to extend the PATH variable to C:\opencv\x64\vc16\bin. [待校准@7697]

Install dependencies [待校准@7698]

There are a few dependencies not available in the Chocolatey package database. In order to ease the manual installation process, we provide the necessary Chocolatey packages. [待校准@7699]

As some chocolatey packages rely on it, we start by installing CMake [待校准@7700]

choco install -y cmake

You will need to append the CMake bin folder C:\Program Files\CMake\bin to your PATH. [待校准@7701]

Please download these packages from this GitHub repository. [待校准@7702]

  • asio.1.12.1.nupkg [待校准@7703]

  • bullet.3.17.nupkg [待校准@7704]

  • cunit.2.1.3.nupkg [待校准@7705]

  • eigen-3.3.4.nupkg [待校准@7706]

  • tinyxml-usestl.2.6.2.nupkg [待校准@7707]

  • tinyxml2.6.0.0.nupkg [待校准@7708]

Once these packages are downloaded, open an administrative shell and execute the following command: [待校准@7709]

choco install -y -s <PATH\TO\DOWNLOADS\> asio cunit eigen tinyxml-usestl tinyxml2 bullet

Please replace <PATH\TO\DOWNLOADS> with the folder you downloaded the packages to. [待校准@7710]

First upgrade pip and setuptools: [待校准@7711]

python -m pip install -U pip setuptools==59.6.0

Now install some additional python dependencies: [待校准@7712]

python -m pip install -U catkin_pkg cryptography empy importlib-metadata lark==1.1.1 lxml matplotlib netifaces numpy opencv-python PyQt5 pillow psutil pycairo pydot pyparsing==2.4.7 pyyaml rosdistro

Install Qt5 [待校准@7713]

Download the 5.12.X offline installer from Qt's website. Run the installer. Make sure to select the MSVC 2017 64-bit component under the Qt -> Qt 5.12.12 tree. [待校准@7714]

Finally, in an administrator cmd.exe window set these environment variables. The commands below assume you installed it to the default location of C:\Qt. [待校准@7715]

setx /m Qt5_DIR C:\Qt\Qt5.12.12\5.12.12\msvc2017_64
setx /m QT_QPA_PLATFORM_PLUGIN_PATH C:\Qt\Qt5.12.12\5.12.12\msvc2017_64\plugins\platforms

注解

This path might change based on the installed MSVC version, the directory Qt was installed to, and the version of Qt installed. [待校准@7716]

RQt dependencies [待校准@7717]

To run rqt_graph you need to download and install Graphviz. The installer will ask if to add graphviz to PATH, choose to either add it to the current user or all users. [待校准@7718]

Install additional DDS implementations (optional) [待校准@7588]

If you would like to use another DDS or RTPS vendor besides the default, Fast DDS, you can find instructions here. [待校准@7719]

Downloading ROS 2 [待校准@7624]

  • Go to the releases page: https://github.com/ros2/ros2/releases [待校准@7992]

  • Download the latest package for Windows, e.g., ros2-humble-*-windows-release-amd64.zip. [待校准@7993]

注解

There may be more than one binary download option which might cause the file name to differ. [待校准@7994]

注解

To install debug libraries for ROS 2, see Extra Stuff for Debug. Then continue on with downloading ros2-package-windows-debug-AMD64.zip. [待校准@7995]

  • Unpack the zip file somewhere (we'll assume C:\dev\ros2_humble). [待校准@7996]

Environment setup [待校准@7594]

Start a command shell and source the ROS 2 setup file to set up the workspace: [待校准@7742]

call C:\dev\ros2_humble\local_setup.bat

It is normal that the previous command, if nothing else went wrong, outputs "The system cannot find the path specified." exactly once. [待校准@7744]

Try some examples [待校准@7597]

In a command shell, set up the ROS 2 environment as described above and then run a C++ talker: [待校准@7997]

ros2 run demo_nodes_cpp talker

Start another command shell and run a Python listener: [待校准@7998]

ros2 run demo_nodes_py listener

You should see the talker saying that it's Publishing messages and the listener saying I heard those messages. This verifies both the C++ and Python APIs are working properly. Hooray! [待校准@7600]

Next steps after installing [待校准@7601]

Continue with the tutorials and demos to configure your environment, create your own workspace and packages, and learn ROS 2 core concepts. [待校准@7602]

Additional RMW implementations (optional) [待校准@7603]

The default middleware that ROS 2 uses is Fast DDS, but the middleware (RMW) can be replaced at runtime. See the guide on how to work with multiple RMWs. [待校准@7952]

Troubleshooting techniques can be found here. [待校准@7769]

Uninstall [待校准@7613]

  1. If you installed your workspace with colcon as instructed above, "uninstalling" could be just a matter of opening a new terminal and not sourcing the workspace's setup file. This way, your environment will behave as though there is no Humble install on your system. [待校准@7614]

  2. If you're also trying to free up space, you can delete the entire workspace directory with: [待校准@7615]

    rmdir /s /q \ros2_humble

Extra Stuff for Debug [待校准@7999]

To download the ROS 2 debug libraries you'll need to download ros2-humble-*-windows-debug-AMD64.zip. Please note that debug libraries require some more additional configuration/setup to work as given below. [待校准@8000]

Python installation may require modification to enable debugging symbols and debug binaries: [待校准@8001]

  • Search in windows Search Bar and open Apps and Features. [待校准@8002]

  • Search for the installed Python version. [待校准@8003]

  • Click Modify. [待校准@8004]

  • Click Next to go to Advanced Options. [待校准@8005]

  • Make sure Download debugging symbols and Download debug binaries are checked. [待校准@8006]

  • Click Install. [待校准@8007]

(Alternative) ROS 2 Build Installation from aka.ms/ros [待校准@8008]

https://aka.ms/ros project hosts ROS 2 builds against the release snapshots. This section explains how to install ROS 2 from this channel. [待校准@8009]

Install ROS 2 builds [待校准@8010]

In an administrative command prompt, run the following commands. [待校准@8011]

mkdir c:\opt\chocolatey
set PYTHONNOUSERSITE=1
set ChocolateyInstall=c:\opt\chocolatey
choco source add -n=ros-win -s="https://aka.ms/ros/public" --priority=1
choco upgrade ros-foxy-desktop -y --execution-timeout=0

Environment setup [待校准@7594]

Start an administrative command prompt and source the ROS 2 setup file to set up the workspace: [待校准@8012]

call C:\opt\ros\foxy\x64\local_setup.bat

Stay up-to-date [待校准@8013]

To keep up-to-date with the latest builds, run: [待校准@8014]

set ChocolateyInstall=c:\opt\chocolatey
choco upgrade all -y --execution-timeout=0

If you want to completely remove the environment downloaded above, run this command: [待校准@8015]

rmdir /s /q C:\opt\
