Visualizing ROS 2 data with Foxglove Studio [待校准@10852]

Foxglove Studio is an open source visualization and debugging tool for your robotics data. [待校准@10853]

It is available in a variety of ways to make development as convenient as possible – it can be run as a standalone desktop app, accessed via your browser, or even self-hosted on your own domain. [待校准@10854]

View the source code on GitHub. [待校准@10855]

Installation [待校准@10856]

To use the web app, simply open Google Chrome and navigate to studio.foxglove.dev. [待校准@10857]

To use the desktop app for Linux, macOS, or Windows, download it directly from the Foxglove Studio website. [待校准@10858]

Connect to a data source [待校准@10859]

On opening Foxglove Studio, you will see a dialog with a list of all possible data sources. [待校准@10860]

To connect to your ROS 2 stack, click "Open connection", select the "Rosbridge (ROS 1 & 2)" tab, and configure your "WebSocket URL". [待校准@10861]

You could also drag-and-drop any local ROS 2 .db3 files directly into the application to load them for playback. [待校准@10862]

注解

In order to load custom message definitions in your ROS 2 files, try converting them to the MCAP file format. [待校准@10863]

Check out the Foxglove Studio docs for more detailed instructions. [待校准@10864]

Building layouts with panels [待校准@10865]

Panels are modular visualization interfaces that can be configured and arranged into Studio layouts. You can also save your layouts for future use, for your own personal reference or with your larger robotics team. [待校准@10866]

Find the full list of available panels in the sidebar's "Add panel" tab. [待校准@10867]

We've highlighted some particularly useful ones below: [待校准@10868]

1 3D: Display visualization markers in a 3D scene [待校准@10869]

Publish marker messages to add primitive shapes (arrows, spheres, etc.) and more complex visualizations (occupancy grids, point clouds, etc.) to your 3D panel's scene. [待校准@10870]

Choose the topics you want to display via the topic picker on the left, and configure each topic's visualization settings in the "Edit topic settings" menu. [待校准@10871]

Reference the docs for a full list of supported message types and some useful user interactions. [待校准@10873]

2 Diagnostics: Filter and sort diagnostics messages [待校准@10874]

Display the status of seen nodes (i.e. stale, error, warn, or OK) from topics with a diagnostic_msgs/msg/DiagnosticArray datatype in a running feed, and display the diagnostics data for a given diagnostic_name/hardware_id. [待校准@10875]

Reference the docs for more details. [待校准@10877]

3 Image: View camera feed images [待校准@10878]

Select a sensor_msgs/msg/Image or sensor_msgs/msg/CompressedImage topic to display. [待校准@10879]

Reference the docs for more details. [待校准@10881]

4 Log: View log messages [待校准@10882]

To view rcl_interfaces/msg/Log messages live, use the desktop app to connect to your running ROS stack. To view rcl_interfaces/msg/Log messages from a pre-recorded data file, you can drag-and-drop your file into either the web or desktop app. [待校准@10883]

Next, add a Log panel to your layout. If you've connected to your ROS stack correctly, you should now see a list of your log messages, with the ability to filter them by node name or severity level. [待校准@10884]

Reference the docs for more details. [待校准@10885]

5 Plot: Plot arbitrary values over time [待校准@10886]

Plot arbitrary values from your topics' message paths over playback time. [待校准@10887]

Specify the topic values you want to plot along the y-axis. For the x-axis, choose between plotting the y-axis value's timestamp, element index, or another custom topic message path. [待校准@10888]

Reference the docs for more details. [待校准@10890]

6 Raw Messages: View incoming topic messages [待校准@10891]

Display incoming topic data in an easy-to-read collapsible JSON tree format. [待校准@10892]

Reference the docs for more details. [待校准@10894]

7 Teleop: Teleoperate your robot [待校准@10895]

Teleoperate your physical robot by publishing geometry_msgs/msg/Twist messages on a given topic back to your live ROS stack. [待校准@10896]

Reference the docs for more details. [待校准@10898]

8 URDF Viewer: View and manipulate your URDF model [待校准@10899]

To visualize and control your robot model in Foxglove Studio, open the web or desktop application and add a URDF Viewer panel to your layout. Then, drag and drop your URDF file into that panel to visualize your robot model. [待校准@10900]

Select any topic publishing a JointState message to update the visualization based on the published joint states (defaults to /joint_states). [待校准@10901]

Toggle to "Manual joint control" to set joint positions using the provided controls. [待校准@10902]

Reference the docs for more details. [待校准@10904]

Other basic actions [待校准@10905]

1 View your ROS graph [待校准@10906]

Using the desktop app, connect to your running ROS stack. Next, add a Topic Graph panel to your layout. If you've connected to your ROS stack correctly, you should now see a computational graph of your ROS nodes, topics, and services in that panel. Use the controls on the right side of the panel to select which topics to display or to toggle services. [待校准@10907]

2 View and edit your ROS params [待校准@10908]

Using the desktop app, connect to your running ROS stack. Next, add a Parameters panel to your layout. If you've connected to your ROS stack correctly, you should now see a live view of your current rosparams. You can edit these parameter values to publish rosparam updates back to your ROS stack. [待校准@10909]

3 Publish messages back to your live ROS stack [待校准@10910]

Using the desktop app, connect to your running ROS stack. Next, add a Publish panel to your layout. [待校准@10911]

Specify the topic you want to publish on to infer its datatype and populate the text field with a JSON message template. [待校准@10912]

Selecting a datatype in the dropdown of common ROS datatypes will also populate the text field with a JSON message template. [待校准@10913]

Edit the template to customize your message before hitting "Publish". [待校准@10914]
