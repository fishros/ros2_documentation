Migrating Launch Files [校准]

目录 []

  • 背景 [15823]

  • Migrating tags [校准]

  • ROS 2中的新标签 [15894]

  • 替换了`include`标签 [15903]

  • 替换 [15907]

  • 类型推断规则 [15842]

While launch files in ROS 1 are always specified using .xml files, ROS 2 supports Python scripts to enable more flexibility (see launch package) as well as XML and YAML files. [校准]

This guide describes how to write ROS 2 XML launch files for an easy migration from ROS 1. [校准]

背景 [15823]

A description of the ROS 2 launch system and its Python API can be found in Launch System tutorial. [校准]

Migrating tags [校准]

启动 [15826]

  • 在ROS 1中可用。 [15827]

  • launch 是任何ROS 2启动XML文件的根元素。 [15828]

节点 [15829]

  • 在ROS 1中可用。 [15830]

  • 启动一个新的节点。 [15831]

  • 与ROS 1的不同之处： [15832]

    • type 属性现在为 exec。 [15833]

    • ns 属性现在为 namespace。 [15834]

    • 以下属性不可用：machine，respawn_delay，clear_params。 [15835]

示例 [15836]

<launch>
   <node pkg="demo_nodes_cpp" exec="talker"/>
   <node pkg="demo_nodes_cpp" exec="listener"/>
</launch>

param [15837]

  • 在 ROS 1 中可用. [15838]

  • 用于将参数传递给节点。 [15839]

  • There's no global parameter concept in ROS 2. For that reason, it can only be used nested in a node tag. Some attributes aren't supported in ROS 2: type, textfile, binfile, executable. [校准]

  • The command attribute is now value="$(command '...' )". [校准]

示例 [15836]

<launch>
   <node pkg="demo_nodes_cpp" exec="parameter_event">
      <param name="foo" value="5"/>
   </node>
</launch>

类型推断规则 [15842]

以下是一些编写参数的示例: [15843]

<node pkg="my_package" exec="my_executable" name="my_node">
   <!--A string parameter with value "1"-->
   <param name="a_string" value="'1'"/>
   <!--A integer parameter with value 1-->
   <param name="an_int" value="1"/>
   <!--A float parameter with value 1.0-->
   <param name="a_float" value="1.0"/>
   <!--A string parameter with value "asd"-->
   <param name="another_string" value="asd"/>
   <!--Another string parameter, with value "asd"-->
   <param name="string_with_same_value_as_above" value="'asd'"/>
   <!--Another string parameter, with value "'asd'"-->
   <param name="quoted_string" value="\'asd\'"/>
   <!--A list of strings, with value ["asd", "bsd", "csd"]-->
   <param name="list_of_strings" value="asd, bsd, csd" value-sep=", "/>
   <!--A list of ints, with value [1, 2, 3]-->
   <param name="list_of_ints" value="1,2,3" value-sep=","/>
   <!--Another list of strings, with value ["1", "2", "3"]-->
   <param name="another_list_of_strings" value="'1';'2';'3'" value-sep=";"/>
   <!--A list of strings using an strange separator, with value ["1", "2", "3"]-->
   <param name="strange_separator" value="'1'//'2'//'3'" value-sep="//"/>
</node>

参数分组 [15844]

在ROS 2中，允许嵌套``param``标签。例如： [15845]

<node pkg="my_package" exec="my_executable" name="my_node" ns="/an_absoulute_ns">
   <param name="group1">
      <param name="group2">
         <param name="my_param" value="1"/>
      </param>
      <param name="another_param" value="2"/>
   </param>
</node>

这将创建两个参数： [15846]

  • 一个值为``1``的``group1.group2.my_param``，由节点``/an_absolute_ns/my_node``托管。 [15847]

  • 节点 /an_absolute_ns/my_node 托管的 group1.another_param 的值为 2。 [15848]

也可以使用完整的参数名称： [15849]

<node pkg="my_package" exec="my_executable" name="my_node" ns="/an_absoulute_ns">
   <param name="group1.group2.my_param" value="1"/>
   <param name="group1.another_param" value="2"/>
</node>

rosparam [15850]

  • 在 ROS 1 中可用 <https://wiki.ros.org/roslaunch/XML/rosparam>`__。 [15851]

  • 从yaml文件中加载参数。 [15852]

  • 已被``param``标签中的``from``属性替代。 [15853]

示例 [15836]

<node pkg="my_package" exec="my_executable" name="my_node" ns="/an_absoulute_ns">
   <param from="/path/to/file"/>
</node>

remap [15854]

  • `在ROS 1中可用<https://wiki.ros.org/roslaunch/XML/remap>`__。 [15855]

  • 用于将重新映射规则传递给节点。 [15856]

  • 它只能在``node``标签内使用。 [15857]

示例 [15836]

<launch>
   <node pkg="demo_nodes_cpp" exec="talker">
      <remap from="chatter" to="my_topic"/>
   </node>
   <node pkg="demo_nodes_cpp" exec="listener">
      <remap from="chatter" to="my_topic"/>
   </node>
</launch>

包括 [15858]

  • 在ROS 1中可用。 [15859]

  • 允许包含另一个启动文件。 [15860]

  • 与ROS 1的不同之处： [15832]

    • 在ROS 1中可用，所包含的内容是有作用域的。在ROS 2中，它没有作用域。使用``group``标签来嵌套包含文件以进行作用域控制。 [15861]

    • 不支持``ns``属性。参考``push_ros_namespace``标签的示例以获取解决方法。 [15862]

    • arg tags nested in an include tag don't support conditionals (if, unless) or the description attribute. [校准]

    • 不支持嵌套``env``标签。可以使用``set_env``和``unset_env``代替。 [15864]

    • 不支持``clear_params``和``pass_all_args``属性。 [15865]

示例 [15866]

请参阅`替换 include 标签`_。 [15867]

arg [15868]

  • 在ROS 1中可用。 [15869]

  • arg 用于声明一个启动参数，或者在使用 include 标签时传递参数。 [15870]

  • 与ROS 1的不同之处： [15832]

    • 不允许使用 value 属性。请使用 let 标签代替。 [15871]

    • 现在，doc 变为 description。 [15872]

    • When nested within an include tag, if, unless, and description attributes aren't allowed. [校准]

示例 [15836]

<launch>
   <arg name="topic_name" default="chatter"/>
   <node pkg="demo_nodes_cpp" exec="talker">
      <remap from="chatter" to="$(var topic_name)"/>
   </node>
   <node pkg="demo_nodes_cpp" exec="listener">
      <remap from="chatter" to="$(var topic_name)"/>
   </node>
</launch>

向启动文件传递参数 [15874]

在上述 XML 启动文件中，topic_name 默认为 chatter，但可以通过命令行进行配置。假设上述启动配置保存在名为 mylaunch.xml 的文件中，可以通过以下命令启动并使用不同的主题名称： [15875]

ros2 launch mylaunch.xml topic_name:=custom_topic_name

There is some additional information about passing command-line arguments in Using Substitutions. [校准]

env [15877]

  • 在 ROS 1 中可用。 [15878]

  • 设置环境变量。 [15879]

  • 已被替换为 env、set_env 和 unset_env： [15880]

    • env 只能嵌套在 node 或 executable 标签中。不支持 if 和 unless 标签。 [15881]

    • set_env 可以嵌套在根标签 launch 或 group 标签中。它接受与 env 相同的属性，还支持 if 和 unless 标签。 [15882]

    • unset_env 取消设置环境变量。它接受一个 name 属性和条件语句。 [15883]

示例 [15836]

<launch>
   <set_env name="MY_ENV_VAR" value="MY_VALUE" if="CONDITION_A"/>
   <set_env name="ANOTHER_ENV_VAR" value="ANOTHER_VALUE" unless="CONDITION_B"/>
   <set_env name="SOME_ENV_VAR" value="SOME_VALUE"/>
   <node pkg="MY_PACKAGE" exec="MY_EXECUTABLE" name="MY_NODE">
      <env name="NODE_ENV_VAR" value="SOME_VALUE"/>
   </node>
   <unset_env name="MY_ENV_VAR" if="CONDITION_A"/>
   <node pkg="ANOTHER_PACKAGE" exec="ANOTHER_EXECUTABLE" name="ANOTHER_NODE"/>
   <unset_env name="ANOTHER_ENV_VAR" unless="CONDITION_B"/>
   <unset_env name="SOME_ENV_VAR"/>
</launch>

组 [15884]

  • 在ROS 1中可用。 [15885]

  • 允许限制启动配置的范围。通常与``let``、``include``和``push_ros_namespace``标签一起使用。 [15886]

  • 与ROS 1的不同之处： [15832]

    • 没有``ns``属性。可以使用新的``push_ros_namespace``标签作为解决方法。 [15887]

    • clear_params 属性不可用。 [15888]

    • 它不接受 remap 和 param 标签作为子节点。 [15889]

示例 [15836]

launch-prefix 配置会影响 executable 和 node 标签的操作。如果 use_time_prefix_in_talker 参数为 1，此示例将使用 time 作为前缀，仅适用于 talker。 [15890]

<launch>
   <arg name="use_time_prefix_in_talker" default="0"/>
   <group>
      <let name="launch-prefix" value="time" if="$(var use_time_prefix_in_talker)"/>
      <node pkg="demo_nodes_cpp" exec="talker"/>
   </group>
   <node pkg="demo_nodes_cpp" exec="listener"/>
</launch>

machine [15891]

目前不支持。 [15892]

测试 [15893]

目前不支持。 [15892]

ROS 2中的新标签 [15894]

set_env和unset_env [15895]

查看 env 标签说明。 [15896]

push_ros_namespace [15897]

include 和 group 标签不接受 ns 属性。可以使用以下操作作为解决方法： [15898]

<!-Other tags-->
<group>
   <push_ros_namespace namespace="my_ns"/>
   <!--Nodes here are namespaced with "my_ns".-->
   <!--If there is an include action here, its nodes will also be namespaced.-->
   <push_ros_namespace namespace="another_ns"/>
   <!--Nodes here are namespaced with "another_ns/my_ns".-->
   <push_ros_namespace namespace="/absolute_ns"/>
   <!--Nodes here are namespaced with "/absolute_ns".-->
   <!--The following node receives an absolute namespace, so it will ignore the others previously pushed.-->
   <!--The full path of the node will be /asd/my_node.-->
   <node pkg="my_pkg" exec="my_executable" name="my_node" ns="/asd"/>
</group>
<!--Nodes outside the group action won't be namespaced.-->
<!-Other tags-->

let [15899]

使用`value`属性替代了``arg``标签。 [15900]

<let name="foo" value="asd"/>

可执行文件 [15901]

允许运行任何可执行文件。 [15902]

示例 [15836]

<executable cmd="ls -las" cwd="/var/log" name="my_exec" launch-prefix="something" output="screen" shell="true">
   <env name="LD_LIBRARY" value="/lib/some.so"/>
</executable>

替换了`include`标签 [15903]

为了在ROS 1中像**命名空间**一样包含启动文件，则必须将``include``标签嵌套在``group``标签中。 [15904]

<group>
   <include file="another_launch_file"/>
</group>

然后，不再使用``ns``属性，而是添加``push_ros_namespace``操作标签来指定命名空间： [15905]

<group>
   <push_ros_namespace namespace="my_ns"/>
   <include file="another_launch_file"/>
</group>

只有在指定命名空间时才需要将``include``标签嵌套在``group``标签下。 [15906]

替换 [15907]

关于ROS 1的替代方案的文档可以在 roslaunch XML wiki 中找到。替代语法没有改变，仍然遵循 $(substitution-name arg1 arg2 ...) 的模式。然而，与ROS 1相比，有一些变化： [15908]

  • env 和 optenv 标签已被 env 标签替代。如果环境变量不存在，$(env <NAME>) 将失败。$(env <NAME> '') 与ROS 1的 $(optenv <NAME>) 功能相同。$(env <NAME> <DEFAULT>) 与ROS 1的 $(env <NAME> <DEFAULT>) 或 $(optenv <NAME> <DEFAULT>) 功能相同。 [15909]

  • find 被 find-pkg-share 替代（用于替代已安装包的共享目录）。或者，find-pkg-prefix 将返回已安装包的根目录。 [15910]

  • 新增了一个 exec-in-pkg 替代。例如：$(exec-in-pkg <package_name> <exec_name>)。 [15911]

  • 有一个新的 find-exec 替代方式。 [15912]

  • arg 已被替换为 var。它查看使用 arg 或 let 标签定义的配置。 [15913]

  • eval and dirname substitutions require escape characters for string values, e.g. if="$(eval '\'$(var variable)\' == \'val1\'')". [校准]

  • 不支持 anon 替代方式。 [15915]

类型推断规则 [15842]

在``param``标签的``Type inference rules``子节中显示的规则适用于任何属性。例如： [15916]

<!--Setting a string value to an attribute expecting an int will raise an error.-->
<tag1 attr-expecting-an-int="'1'"/>
<!--Correct version.-->
<tag1 attr-expecting-an-int="1"/>
<!--Setting an integer in an attribute expecting a string will raise an error.-->
<tag2 attr-expecting-a-str="1"/>
<!--Correct version.-->
<tag2 attr-expecting-a-str="'1'"/>
<!--Setting a list of strings in an attribute expecting a string will raise an error.-->
<tag3 attr-expecting-a-str="asd, bsd" str-attr-sep=", "/>
<!--Correct version.-->
<tag3 attr-expecting-a-str="don't use a separator"/>

有些属性接受多个类型，例如``param``标签的``value``属性。通常情况下，类型为``int``（或``float``）的参数也可以接受``str``，后者将在后续进行替换，并尝试转换为``int``（或``float）``。 [15917]
