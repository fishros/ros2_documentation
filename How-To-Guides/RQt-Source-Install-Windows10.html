Building RQt from source on Windows 10 [待校准@10460]

This page provides specific information to building RQt from source on Windows. Follow these instructions before proceeding with the RQt Source Install page. [待校准@10461]

If you have not done so, follow the ROS 2 Windows Development Setup guide before continuing. [待校准@10462]

System Requirements [待校准@10429]

  • Windows 10 [待校准@10463]

  • Visual Studio 15.7.6 [待校准@10464]

Currently Visual Studio 15.8 fails to build ROS 2 (see issue). Older versions of VS can be found here. [待校准@10465]

Dependencies [待校准@10455]

The primary dependencies of the RQt package are sip and PyQt5. PySide2 may be supported in the future. Even though they are provided through PyPI and chocolatey, you must install them by source to get compatible versions. [待校准@10466]

Install sip by source [待校准@10467]

Download from https://www.riverbankcomputing.com/software/sip/download [待校准@10468]

Run the x64 Native Tools Command Prompt as Administrator, and cd to the uncompressed source directory. [待校准@10469]

Run: [待校准@10470]

python3 configure.py
nmake
nmake install

If python3 is installed on your system as python, be sure to use that program name instead. [待校准@10471]

Install PyQt5 by source [待校准@10472]

Download from https://www.riverbankcomputing.com/software/pyqt/download5 [待校准@10473]

Run the x64 Native Tools Command Prompt as Administrator, and cd to the uncompressed source directory. I ran into trouble with Qt 5.11.3 and PyQt5 compiling QtNfc, but it can be easily disabled. [待校准@10474]

python3 configure.py --disable QtNfc
nmake
nmake install

Test that it works [待校准@10475]

If install occurred without failure, try the commands below. They should run without issue and you should see 4.19.13 as your sip.exe version. [待校准@10476]

sip -V
python3 -c "from PyQt5 import QtCore"

Other dependencies [待校准@10477]

Install GraphViz from https://graphviz.gitlab.io/_pages/Download/Download_windows.html. [待校准@10478]

Install pydot and pyparsing: [待校准@10479]

pip3 install pydot pyparsing

PyGraphViz is a test dependency of qt_dotgraph, but it is currently unsupported on Windows and building by source is not straight forward. Manually merging this patch is the currently recommended solution (not verified): pygraphviz patch [待校准@10480]

Install RQt by source [待校准@10458]

Continue with the RQt source install page. [待校准@10459]
