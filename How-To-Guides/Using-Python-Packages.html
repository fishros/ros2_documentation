Using Python Packages with ROS 2 [待校准@10716]

Goal: Explain how to interoperate with other Python packages from the ROS 2 ecosystem. [待校准@10717]

Contents [待校准@10051]

  • Installing via rosdep [待校准@10719]

  • Installing via a package manager [待校准@10726]

  • Installing via a virtual environment [待校准@10730]

注解

A cautionary note, if you intended to use pre-packaged binaries (either deb files, or the “fat” binary distributions), the Python interpreter must match what was used to build the original binaries. If you intend to use something like virtualenv or pipenv, make sure to use the system interpreter. If you use something like conda, it is very likely that the interpreter will not match the system interpreter and will be incompatible with ROS 2 binaries. [待校准@10718]

Installing via rosdep [待校准@10719]

The fastest way to include third-party python packages is to use their corresponding rosdep keys, if available. rosdep keys can be checked via: [待校准@10720]

  • https://github.com/ros/rosdistro/blob/master/rosdep/base.yaml [待校准@10721]

  • https://github.com/ros/rosdistro/blob/master/rosdep/python.yaml [待校准@10722]

These rosdep keys can be added to your package.xml file, which indicates to the build system that your package (and dependent packages) depend on those keys. In a new workspace, you can also quickly install all rosdep keys with: [待校准@10723]

rosdep install -yr ./path/to/your/workspace

If there aren’t currently rosdep keys for the package that you are interested in, it is possible to add them by following the rosdep key contribution guide. [待校准@10724]

To learn more about the rosdep tool and how it works, consult the rosdep documentation. [待校准@10725]

Installing via a package manager [待校准@10726]

If you don’t want to make a rosdep key, but the package is available in your system package manager (eg apt), you can install and use the package that way: [待校准@10727]

sudo apt install python3-serial

If the package is available on The Python Package Index (PyPI) and you want to install globally on your system: [待校准@10728]

python3 -m pip install -U pyserial

If the package is available on PyPI and you want to install locally to your user: [待校准@10729]

python3 -m pip install -U --user pyserial

Installing via a virtual environment [待校准@10730]

First, create a Colcon workspace: [待校准@10731]

mkdir -p ~/colcon_venv/src
cd ~/colcon_venv/

Then setup your virtual environment: [待校准@10732]

# Make a virtual env and activate it
virtualenv -p python3 ./venv
source ./venv/bin/activate
# Make sure that colcon doesn’t try to build the venv
touch ./venv/COLCON_IGNORE

Next, install the Python packages that you want in your virtual environment: [待校准@10733]

python3 -m pip install gtsam pyserial… etc

Now you can build your workspace and run your python node that depends on packages installed in your virtual environment. [待校准@10734]

# Source Humble and build
source /opt/ros/humble/setup.bash
colcon build

注解

If you want to release your package using Bloom, you should add the packages you require to rosdep, see the rosdep key contribution guide. [待校准@10735]
