Changes since the Dashing release [待校准@14525]

geometry_msgs [待校准@14526]

The geometry_msgs/msg/Quaternion.msg interface now default initializes to a valid quaternion, with the following values: [待校准@14527]

\[\begin{split}x = 0 \\ y = 0 \\ z = 0 \\ w = 1 `[待校准@14528] <http://dev.ros2.fishros.com/calibpage/#/home?msgid=14528>`_\end{split}\]

Here is the pull request for more detail: https://github.com/ros2/common_interfaces/pull/74 [待校准@14529]

Static transform broadcasters and listeners now use QoS durability transient_local on the /tf_static topic. Similar to the latched setting in ROS 1, static transforms only need to be published once. New listeners will receive transforms from all static broadcasters that are alive and have published before. All publishers must be updated to use this durability setting or their messages won't be received by transform listeners. See this pull request for more detail: https://github.com/ros2/geometry2/pull/160 [待校准@14530]

rclcpp [待校准@14358]

API Break with get_actual_qos() [待校准@14531]

Introduced in Dashing, the get_actual_qos() method on the PublisherBase and SubscriptionBase previously returned an rmw type, rmw_qos_profile_t, but that made it awkward to reuse with the creation of other entities. Therefore it was updated to return a rclcpp::QoS instead. [待校准@14532]

Existing code will need to use the rclcpp::QoS::get_rmw_qos_profile() method if the rmw profile is still required. For example: [待校准@14533]

void my_func(const rmw_qos_profile_t & rmw_qos);

/* Previously: */
// my_func(some_pub->get_actual_qos());
/* Now: */
my_func(some_pub->get_actual_qos()->get_rmw_qos_profile());

The rationale for breaking this directly rather than doing a tick-tock is that it is a new function and is expected to be used infrequently by users. Also, since only the return type is changing, adding a new function with a different would be to only way to do a deprecation cycle and get_actual_qos() is the most appropriate name, so we would be forced to pick a less obvious name for the method. [待校准@14534]

API Break with Publisher and Subscription Classes [待校准@14535]

In an effort to streamline the construction of Publishers and Subscriptions, the API of the constructors were changed. [待校准@14536]

It would be impossible to support a deprecation cycle, because the old signature takes an rcl type and the new one takes the NodeBaseInterface type so that it can get additional information it now needs, and there's no way to get the additional information needed from just the rcl type. The new signature could possibly be backported if that would help contributors, but since the publishers and subscriptions are almost always created using the factory functions or some other higher level API, we do not expect this to be a problem for most users. [待校准@14537]

Please see the original pr for more detail and comment there if this causes issues: [待校准@14538]

https://github.com/ros2/rclcpp/pull/867 [待校准@14539]

Compiler warning about unused result of add_on_set_parameters_callback [待校准@14540]

Since Eloquent Patch Release 2 (2020-12-04) [待校准@14541]

Users should retain the handle returned by rclcpp::Node::add_on_set_parameters_callback, otherwise their callback may be unregistered. A warning has been added to help identify bugs where the returned handle is not used. [待校准@14542]

https://github.com/ros2/rclcpp/pull/1243 [待校准@14543]

rmw [待校准@14450]

API Break Due to Addition of Publisher and Subscription Options [待校准@14544]

The rmw_create_publisher() method had a new argument added of type const rmw_publisher_options_t *. This new structure holds options (beyond the typesupport, topic name, and QoS) for new publishers. [待校准@14545]

The rmw_create_subscription() method had one argument removed, bool ignore_local_publications, and replaced by the new options of type const rmw_subscription_options_t *. The ignore_local_publications option was moved into the new rmw_subscription_options_t type. [待校准@14546]

In both cases the new argument, which are pointers, may never be null, and so the rmw implementations should check to make sure the options are not null. Additionally, the options should be copied into the corresponding rmw structure. [待校准@14547]

See this pull request, and the associated pull requests for more details: [待校准@14548]

https://github.com/ros2/rmw/pull/187 [待校准@14549]

ros2cli [待校准@14550]

ros2msg and ros2srv deprecated [待校准@14551]

The CLI tools ros2msg and ros2srv are deprecated. They have been replaced by the tool ros2interface, which also supports action and IDL interfaces. You can run ros2 interface --help for usage. [待校准@14552]

ros2node [待校准@14553]

Service clients have been added to ros2node info. As part of that change the Python function ros2node.api.get_service_info has been renamed to ros2node.api.get_service_server_info. [待校准@14554]

rviz [待校准@14477]

Renamed '2D Nav Goal' tool [待校准@14555]

The tool was renamed to '2D Goal Pose' and the default topic was changed from /move_base_simple/goal to /goal_pose. [待校准@14556]

Here is the related pull request: [待校准@14557]

https://github.com/ros2/rviz/pull/455 [待校准@14558]

TF2 Buffer [待校准@14559]

TF2 buffers now have to be given a timer interface. [待校准@14560]

If a timer interface is not given, an exception will be thrown. [待校准@14561]

For example: [待校准@14562]

tf = std::make_shared<tf2_ros::Buffer>(get_clock());
// The next two lines are new in Eloquent
auto timer_interface = std::make_shared<tf2_ros::CreateTimerROS>(
  this->get_node_base_interface(),
  this->get_node_timers_interface());
tf->setCreateTimerInterface(timer_interface);
// Pass the Buffer to the TransformListener as before
transform_listener = std::make_shared<tf2_ros::TransformListener>(*tf);

rcl [待校准@14563]

ROS command line argument changes [待校准@14564]

To cope with an increasingly complex interface, with a now extended set of configuration options, ROS CLI syntax has been changed. As an example, a command line using Dashing syntax like: [待校准@14565]

ros2 run some_package some_node foo:=bar __params:=/path/to/params.yaml __log_level:=WARN --user-flag

is written using Eloquent (and onwards) syntax as: [待校准@14566]

ros2 run some_package some_node --ros-args --remap foo:=bar --params-file /path/to/params.yaml --log-level WARN -- --user-flag

This explicit syntax affords new features, like single parameter assignment --param name:=value. For further reference and rationale, check the ROS command line arguments design document. [待校准@14567]

警告

Former syntax has been deprecated and is due for removal in the next release. [待校准@14568]

Known Issues [待校准@14201]

  • [ros2/rosidl#402] find_package(PCL) interferes with ROS interface generation. Workaround: invoke find_package(PCL) after rosidl_generate_interfaces(). [待校准@14569]

  • [ros2/rclcpp#893] rclcpp::Context is not destroyed because of a reference cycle with rclcpp::GraphListener. This causes a memory leak. A fix has not been backported because of the risk of breaking ABI. [待校准@14486]
