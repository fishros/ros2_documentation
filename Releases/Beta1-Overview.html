Supported Platforms [待校准@8234]

We support ROS 2 Beta 1 on three platforms: Ubuntu 16.04 (Xenial), Mac OS X 10.11 (El Capitan), and Windows 8.1 and 10. We provide both binary packages and instructions for how to compile from source for all 3 platforms. [待校准@8235]

Features [待校准@8236]

Improvements since Alpha 8 release [待校准@8237]

  • Support for node composition at compile, link, or runtime. [待校准@8238]

  • A standard lifecycle for managed nodes. [待校准@8239]

  • Improved support for Quality of Service tuning and tests. [待校准@8240]

  • New and updated design documents [待校准@8241]

  • More tutorials and examples [待校准@8242]

  • Bridging services to / from ROS 1 (in addition to topics) [待校准@8243]

Selected features from previous Alpha releases [待校准@8244]

For the complete list, see earlier release notes. [待校准@8245]

  • C++ and Python implementations of ROS 2 client libraries including APIs for: [待校准@8246]

    • Publishing and subscribing to ROS topics [待校准@8247]

    • Requesting and replying ROS services (synchronous (C++ only) and asynchronous) [待校准@8248]

    • Getting and setting ROS parameters (C++ only, synchronous and asynchronous) [待校准@8249]

    • Timer callbacks [待校准@8250]

    • Support for interoperability between multiple DDS/RTPS implementations [待校准@8251]

    • eProsima Fast RTPS is our default implementation, and is included in the binary packages [待校准@8252]

    • RTI Connext is supported: build from source to try it out [待校准@8253]

    • We initially supported PrismTech OpenSplice but eventually decided to drop it [待校准@8254]

  • A graph API for network events [待校准@8255]

  • Distributed discovery [待校准@8256]

  • Realtime safe code paths for publish and subscribe with compatible DDS implementation (only Connext at the moment) [待校准@8257]

    • Support for custom allocators [待校准@8258]

  • ROS 1 <-> ROS 2 dynamic bridge node [待校准@8259]

  • Executor threading model in C++ [待校准@8260]

  • Extended .msg format with new features: [待校准@8261]

    • Bounded arrays [待校准@8262]

    • Default values [待校准@8263]

Known issues [待校准@8264]

  • We’re tracking issues in various repositories, but the main entry point is the ros2/ros2 issue tracker [待校准@8265]

  • We’d like to highlight a known issue that we are working with eProsima to fix that results in significantly degrated performance for large messages under FastRTPS. This will be observed when running some of the demos with larger image resolutions. [待校准@8266]
