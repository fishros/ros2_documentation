欢迎使用 ROS 2 软件的第一个非测试版发布，命名为 Ardent Apalone！ [14164]

支持的平台 [8234]

ROS 2的这个版本支持三个平台： [14165]

  • Ubuntu 16.04 (Xenial) [14166]

  • Mac macOS 10.12 (Sierra) [14167]

  • Windows 10 [14168]

提供了三个平台的二进制包和源代码编译的说明（参见`安装说明<../../Installation>`以及`文档<https://docs.ros2.org/ardent/>`__）。 [14169]

特性 [8236]

这个ROS 2版本的新功能有： [14170]

  • 分布式发现、发布/订阅、请求/响应通信 [14171]

    • 通过 C API 提供 [14172]

    • 使用不同的供应商实现： [14173]

      • eProsima 的 Fast RTPS 和 ADLINK 的 OpenSplice（来自二进制和源代码） [14174]

      • RTI 的 Connext（仅来自源代码） [14175]

    • 用于处理非理想网络的众多服务质量设置 [14176]

    • DDS 安全支持 (使用 Connext 和 Fast RTPS) [14177]

  • C++ 和 Python 3 客户端库 [14178]

    • 在 C 中共享公共代码，以统一实现 [14179]

    • 执行模型与节点分离，可组合的节点 [14180]

    • 节点特定参数（目前仅限 C++） [14181]

    • 生命周期（目前仅限 C++） [14182]

    • 可选择使用相同的API进行进程内通信（仅限C++） [14183]

  • 消息定义（包括有界数组和字符串以及默认值） [14184]

  • 命令行工具（例如 ros2 run） [14185]

  • rviz 支持几种显示类型（Windows版本可能会在几周后发布） [14186]

  • 基于文件系统的资源索引（无需递归遍历查询信息） [14187]

  • 发布/订阅实时安全的代码路径（仅与兼容的DDS实现兼容） [14188]

  • ROS 1和ROS 2之间的桥梁 [14189]

  • HSR演示 查看Beta 3 [14190]

  • Turtlebot演示 查看Beta 2 [14191]

如需更详细的描述，请参阅`功能 <../../The-ROS2-Project/Features>`页面。 [14192]

自Beta 3发布以来的变化 [14193]

自Beta 3发布以来的改进： [14194]

  • rviz [14195]

  • C++中消息数据结构的不同初始化选项（详见`设计文档 <https://design.ros2.org/articles/generated_interfaces_cpp.html#constructors>`__） [14196]

  • 日志API改进，现在也在演示中使用 [14197]

  • C++中的时间支持，使用不同的时钟 [14198]

  • Python客户端库中的wait-for-service支持 [14199]

  • 对软件包清单文件格式3的 REP 149 进行初步实现 [14200]

已知问题 [14201]

  • 对于像图像演示那样的大型数据，Fast RTPS 的性能较好 [14202]

  • 使用 Connext，不允许具有相同基本名称但不同命名空间的两个主题具有不同的类型（请参阅 问题）。 [8316]

  • 节点名称列表（例如使用 ros2 node list）在某些 rmw 实现中无法正常工作 [14203]

  • 在 Windows 上，Python 启动文件可能在尝试使用 Ctrl-C 中止时挂起（请参阅 问题）。为了继续使用被挂起命令阻塞的 shell，您可能希望使用进程监视器结束挂起的 Python 进程。 [8314]
