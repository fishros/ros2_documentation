Welcome to the first non-beta release of ROS 2 software named Ardent Apalone! [待校准@6640]

Supported Platforms [待校准@710]

This version of ROS 2 is supported on three platforms: [待校准@6641]

  • Ubuntu 16.04 (Xenial) [待校准@6642]

  • Mac macOS 10.12 (Sierra) [待校准@6643]

  • Windows 10 [待校准@6644]

Binary packages as well as instructions for how to compile from source are provided for all 3 platforms (see install instructions as well as documentation). [待校准@6645]

Features [待校准@712]

New features in this ROS 2 release [待校准@6646]

  • Distributed discovery, publish / subscribe, request / response communication [待校准@6647]

    • Provided by a C API [待校准@6648]

    • Implemented using different vendors: [待校准@6649]

      • eProsima's Fast RTPS as well as ADLINK's OpenSplice (from binary and source) [待校准@6650]

      • RTI's Connext (only from source) [待校准@6651]

    • Numerous quality of service settings for handling non-ideal networks [待校准@6652]

    • DDS Security support (with Connext and Fast RTPS) [待校准@6653]

  • C++ and Python 3 client libraries [待校准@6654]

    • Sharing common code in C to unify the implementation [待校准@6655]

    • Execution model separated from the nodes, composable nodes [待校准@6656]

    • Node-specific parameters (only in C++ atm) [待校准@6657]

    • Life cycle (only in C++ atm) [待校准@6658]

    • Optionally intra-process communication using the same API (only in C++) [待校准@6659]

  • Message definitions (with bounded arrays and strings as well as default values) [待校准@6660]

  • Command line tools (e.g. ros2 run) [待校准@6661]

  • rviz with a few display types (the Windows version will likely follow in a few weeks) [待校准@6662]

  • File system-based resource index (querying information without recursive crawling) [待校准@6663]

  • Realtime safe code paths for pub / sub (with compatible DDS implementations only) [待校准@6664]

  • Bridge between ROS 1 and ROS 2 [待校准@6665]

  • HSR demo see Beta 3 [待校准@6666]

  • Turtlebot demo see Beta 2 [待校准@6667]

For a more detailed description please see the Features page. [待校准@6668]

Changes since Beta 3 release [待校准@6669]

Improvements since the Beta 3 release: [待校准@6670]

  • rviz [待校准@6671]

  • Different initialization options for message data structures in C++ (see design doc) [待校准@6672]

  • Logging API improvements, now also used in the demos [待校准@6673]

  • Time support in C++ with different clocks [待校准@6674]

  • wait-for-service support in the Python client library [待校准@6675]

  • Draft implementation of REP 149 specifying format 3 of the package manifest files [待校准@6676]

Known Issues [待校准@6677]

  • Fast RTPS performance with larger data like the image demo [待校准@6678]

  • Using Connext it is currently not allowed for two topics with the same base name but different namespaces to have a different type (see issue). [待校准@792]

  • Listing of node names (e.g. using ros2 node list) does not work across some rmw implementations. [待校准@6679]

  • On Windows Python launch files might hang when trying to abort using Ctrl-C (see issue). In order to continue using the shell which is blocked by the hanging command you might want to end the hanging Python process using the process monitor. [待校准@790]
