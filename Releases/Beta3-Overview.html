支持的平台 [8234]

我们支持三个平台上的ROS 2 Beta 3：Ubuntu 16.04（Xenial）、macOS 10.12（Sierra）和Windows 10。我们提供了三个平台的二进制包和编译源代码的说明（参见`安装说明<../../Installation>`和`文档<https://docs.ros2.org/beta3/>`__）。 [8299]

特性 [8236]

与Beta 2版本相比的改进 [8300]

  • Python中的执行模型，在Python C扩展中修复了许多内存管理问题 [8301]

  • 实验性重写了`ros_control <https://github.com/ros2/ros2_control>`__ [8302]

  • 向用户公开DDS实现特定的符号（适用于Fast RTPS和Connext）（请参阅`示例 <https://github.com/ros2/demos/blob/6363be2efe2fea799d92bc22a66e776b2ca9c5d0/demo_nodes_cpp_native/src/talker.cpp>`__） [8303]

  • 在Python中记录`API <https://github.com/ros2/rclpy/blob/1ef2924ef8e154c0553edf0fdba4840b08b728f8/rclpy/rclpy/logging.py>`__ [8304]

  • 修复了各种包中的多个内存泄漏和竞争条件问题 [8305]

  • 重新添加了对OpenSplice的支持（目前在Linux和Windows上由PrismTech提供） [8306]

  • 使用bloom（不使用补丁）来制作ROS 2发行版 [8307]

新的演示应用程序 [8276]

  • HSR演示 [8308]

    • 使用ROS 2手柄控制器远程控制HSR机器人 [8309]

    • 在HSR上使用Docker容器运行``ros1_bridge``（因为机器人在Ubuntu Trusty上运行ROS 1） [8310]

    • 运行ROS 2开发版的`rviz <https://github.com/ros2/rviz>`__以可视化来自机器人的传感器数据等（参见`视频 <https://vimeo.com/237016358>`__） [8311]

已知问题 [8264]

  • 在 Windows 上，Python 启动文件可能在尝试使用 Ctrl-C 中止时挂起（请参阅 问题）。为了继续使用被挂起命令阻塞的 shell，您可能希望使用进程监视器结束挂起的 Python 进程。 [8314]

  • 目前尚不支持在 MacOS 上使用 OpenSplice。此外，尚未实现 访问本机句柄。 [8315]

  • 使用 Connext，不允许具有相同基本名称但不同命名空间的两个主题具有不同的类型（请参阅 问题）。 [8316]
