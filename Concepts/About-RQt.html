概述 [6823]

RQt 是一个以插件形式实现各种工具和界面的图形用户界面框架。您可以在 RQt 中将所有现有的 GUI 工具作为可停靠窗口运行！这些工具仍然可以以传统的独立方式运行，但是 RQt 可以更轻松地在单个屏幕布局中管理所有不同的窗口。 [7385]

您可以通过以下方式轻松运行任何 RQt 工具/插件： [7386]

rqt

此图形界面允许您选择系统中的任何可用插件。您还可以在独立窗口中运行插件。例如，RQt Python 控制台： [7387]

ros2 run rqt_py_console rqt_py_console

用户可以使用 Python 或 C++ 创建自己的 RQt 插件。要查看系统上可用的 RQt 插件，请运行： [7388]

ros2 pkg list

然后查找以 rqt_ 开头的软件包。 [7389]

RQt组件结构 [7394]

RQt 由两个元包组成： [7395]

  • rqt - 核心基础设施模块。 [7396]

  • rqt_common_plugins - 常用的调试工具。 [7397]

RQt 框架的优势 [7398]

与从头开始构建自己的 GUI 相比： [7399]

  • 提供了标准化的常用 GUI 过程（启动-关闭钩子，恢复先前状态）。 [7400]

  • 可以在单个窗口中停靠多个小部件。 [7401]

  • 轻松将现有的 Qt 小部件转换为 RQt 插件。 [7402]

  • 在 ROS Answers <https://answers.ros.org>`__（ROS 社区问题的网站）上提供支持。 `[7403]

从系统架构的角度来看： [7404]

  • 支持多平台（基本上在 QT 和 ROS 运行的任何地方）和多语言（Python、C++）。 [7405]

  • 可管理的生命周期：使用共享 API 的 RQt 插件使得维护和重用更加容易。 [7406]
