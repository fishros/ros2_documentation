Background [待校准@3072]

Actions are one of the communication types in ROS 2 and are intended for long running tasks. They consist of three parts: a goal, feedback, and a result. [待校准@4115]

Actions are built on topics and services. Their functionality is similar to services, except actions can be canceled. They also provide steady feedback, as opposed to services which return a single response. [待校准@4116]

Actions use a client-server model, similar to the publisher-subscriber model (described in the topics tutorial). An "action client" node sends a goal to an "action server" node that acknowledges the goal and returns a stream of feedback and a result. [待校准@4117]

Prerequisites [待校准@3124]

This tutorial builds off concepts, like nodes and topics, covered in previous tutorials. [待校准@4118]

This tutorial uses the turtlesim package. [待校准@4119]

As always, don't forget to source ROS 2 in every new terminal you open. [待校准@4070]

Tasks [待校准@3282]

1 Setup [待校准@4071]

Start up the two turtlesim nodes, /turtlesim and /teleop_turtle. [待校准@4120]

Open a new terminal and run: [待校准@3132]

ros2 run turtlesim turtlesim_node

Open another terminal and run: [待校准@4073]

ros2 run turtlesim turtle_teleop_key

2 Use actions [待校准@4121]

When you launch the /teleop_turtle node, you will see the following message in your terminal: [待校准@4122]

Use arrow keys to move the turtle.
Use G|B|V|C|D|E|R|T keys to rotate to absolute orientations. 'F' to cancel a rotation.

Let's focus on the second line, which corresponds to an action. (The first instruction corresponds to the "cmd_vel" topic, discussed previously in the topics tutorial.) [待校准@4123]

Notice that the letter keys G|B|V|C|D|E|R|T form a "box" around the F key on a US QWERTY keyboard (if you are not using a QWERTY keyboard, see this link to follow along). Each key's position around F corresponds to that orientation in turtlesim. For example, the E will rotate the turtle's orientation to the upper left corner. [待校准@4124]

Pay attention to the terminal where the /turtlesim node is running. Each time you press one of these keys, you are sending a goal to an action server that is part of the /turtlesim node. The goal is to rotate the turtle to face a particular direction. A message relaying the result of the goal should display once the turtle completes its rotation: [待校准@4125]

[INFO] [turtlesim]: Rotation goal completed successfully

The F key will cancel a goal mid-execution. [待校准@4126]

Try pressing the C key, and then pressing the F key before the turtle can complete its rotation. In the terminal where the /turtlesim node is running, you will see the message: [待校准@4127]

[INFO] [turtlesim]: Rotation goal canceled

Not only can the client-side (your input in the teleop) stop a goal, but the server-side (the /turtlesim node) can as well. When the server-side chooses to stop processing a goal, it is said to "abort" the goal. [待校准@4128]

Try hitting the D key, then the G key before the first rotation can complete. In the terminal where the /turtlesim node is running, you will see the message: [待校准@4129]

[WARN] [turtlesim]: Rotation goal received before a previous goal finished. Aborting previous goal

This action server chose to abort the first goal because it got a new one. It could have chosen something else, like reject the new goal or execute the second goal after the first one finished. Don't assume every action server will choose to abort the current goal when it gets a new one. [待校准@4130]

4 ros2 action list [待校准@4136]

To identify all the actions in the ROS graph, run the command: [待校准@4137]

ros2 action list

Which will return: [待校准@4077]

/turtle1/rotate_absolute

This is the only action in the ROS graph right now. It controls the turtle's rotation, as you saw earlier. You also already know that there is one action client (part of /teleop_turtle) and one action server (part of /turtlesim) for this action from using the ros2 node info <node_name> command. [待校准@4138]

4.1 ros2 action list -t [待校准@4139]

Actions have types, similar to topics and services. To find /turtle1/rotate_absolute's type, run the command: [待校准@4140]

ros2 action list -t

Which will return: [待校准@4077]

/turtle1/rotate_absolute [turtlesim/action/RotateAbsolute]

In brackets to the right of each action name (in this case only /turtle1/rotate_absolute) is the action type, turtlesim/action/RotateAbsolute. You will need this when you want to execute an action from the command line or from code. [待校准@4141]

5 ros2 action info [待校准@4142]

You can further introspect the /turtle1/rotate_absolute action with the command: [待校准@4143]

ros2 action info /turtle1/rotate_absolute

Which will return [待校准@4144]

Action: /turtle1/rotate_absolute
Action clients: 1
    /teleop_turtle
Action servers: 1
    /turtlesim

This tells us what we learned earlier from running ros2 node info on each node: The /teleop_turtle node has an action client and the /turtlesim node has an action server for the /turtle1/rotate_absolute action. [待校准@4145]

6 ros2 interface show [待校准@4146]

One more piece of information you will need before sending or executing an action goal yourself is the structure of the action type. [待校准@4147]

Recall that you identified /turtle1/rotate_absolute's type when running the command ros2 action list -t. Enter the following command with the action type in your terminal: [待校准@4148]

ros2 interface show turtlesim/action/RotateAbsolute

Which will return: [待校准@4077]

# The desired heading in radians
float32 theta
---
# The angular displacement in radians to the starting position
float32 delta
---
# The remaining rotation in radians
float32 remaining

The section of this message above the first --- is the structure (data type and name) of the goal request. The next section is the structure of the result. The last section is the structure of the feedback. [待校准@4149]

7 ros2 action send_goal [待校准@4150]

Now let's send an action goal from the command line with the following syntax: [待校准@4151]

ros2 action send_goal <action_name> <action_type> <values>

<values> need to be in YAML format. [待校准@4152]

Keep an eye on the turtlesim window, and enter the following command into your terminal: [待校准@4153]

Summary [待校准@3353]

Actions are like services that allow you to execute long running tasks, provide regular feedback, and are cancelable. [待校准@4159]

A robot system would likely use actions for navigation. An action goal could tell a robot to travel to a position. While the robot navigates to the position, it can send updates along the way (i.e. feedback), and then a final result message once it's reached its destination. [待校准@4160]

Turtlesim has an action server that action clients can send goals to for rotating turtles. In this tutorial, you introspected that action, /turtle1/rotate_absolute, to get a better idea of what actions are and how they work. [待校准@4161]
