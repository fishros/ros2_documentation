Installation (Windows) [待校准@12493]

Goal: Install the webots_ros2 package and run simulation examples on Windows. [待校准@12494]

Tutorial level: Advanced [待校准@11828]

Time: 10 minutes [待校准@12423]

Background [待校准@11832]

The webots_ros2 package provides an interface between ROS 2 and Webots. It includes several sub-packages, including webots_ros2_driver, which allows ROS nodes to communicate with Webots. Other sub-packages are mainly examples that show multiple possible implementations using the interface. In this tutorial, you are going to install the package and learn how to run one of these examples. [待校准@12495]

Prerequisites [待校准@11884]

It is recommended to understand basic ROS principles covered in the beginner Tutorials [待校准@11819]. In particular, Creating a workspace [待校准@13188] and Creating a package [待校准@13256] are useful prerequisites. [待校准@12425]

Webots is a prerequisite to use the webots_ros2 package. You can follow the installation procedure or build it from sources. [待校准@12496]

Alternatively, you can also let webots_ros2 download Webots automatically. This option appears when you launch an example of the package and no Webots installation is found. [待校准@12497]

Multiple Installations of Webots [待校准@12481]

If you have more than one installation of Webots, ROS 2 will look for Webots at the following locations (in this order): [待校准@12498]

  1. If the ROS2_WEBOTS_HOME environment variable is set, ROS 2 will use the Webots in this folder, regardless of its version. [待校准@12483]

  2. If the WEBOTS_HOME environment variable is set, ROS 2 will use the Webots in this folder, regardless of its version. [待校准@12484]

  3. If none of the previous points is set/installed ROS 2 will look for Webots in the default installation paths for a compatible version: C:\Program Files\Webots. [待校准@12499]

  4. If Webots couldn't be found, webots_ros2 will show a window and offer automatic Webots installation of the last compatible version. [待校准@12500]

Tasks [待校准@12042]

1 Install WSL2 [待校准@12501]

On Windows, WSL (Windows Subsystem for Linux) improves the user experience with ROS 2 compared to native Windows installation, as it runs on a Linux platform. Install WSL with an Ubuntu version which is compatible with your ROS distribution and upgrade to WSL2 following the official Microsoft tutorial. [待校准@12502]

2 Install ROS 2 in WSL [待校准@12503]

Install ROS 2 inside Ubuntu WSL, following Ubuntu (Debian) [待校准@513]. [待校准@12504]

3 Install webots_ros2 [待校准@12450]

You can then either install webots_ros2 from the official released package, or install it from the latest up-to-date sources from Github. [待校准@12505]

The following commands must be run inside the WSL environment. [待校准@12506]

Run the following command in a terminal. [待校准@12489]

sudo apt-get install ros-humble-webots-ros2

Create a ROS 2 workspace with its src directory. [待校准@12456]

mkdir -p ~/ros2_ws/src

Source the ROS 2 environment. [待校准@12457]

source /opt/ros/humble/setup.bash

Retrieve the sources from Github. [待校准@12458]

cd ~/ros2_ws
git clone --recurse-submodules https://github.com/cyberbotics/webots_ros2.git src/webots_ros2

Install the package dependencies. [待校准@12459]

sudo apt install python3-pip python3-rosdep python3-colcon-common-extensions
sudo rosdep init && rosdep update
rosdep install --from-paths src --ignore-src --rosdistro humble

Build the package using colcon. [待校准@12460]

colcon build

Source this workspace. [待校准@12461]

source install/local_setup.bash

4 Launch the webots_ros2_universal_robot example [待校准@12462]

WSL doesn't support hardware acceleration (yet). Therefore, Webots should be started on Windows, while the ROS part is running inside WSL. To do so, the following commands must be run inside the WSL environment. [待校准@12507]

First source the ROS 2 environment, if not done already. [待校准@12466]

source /opt/ros/humble/setup.bash

Setting the WEBOTS_HOME environment variable allows you to start a specific Webots installation (e.g. C:\Program Files\Webots). Use the mount point "/mnt" to refer to a path on native Windows. [待校准@12508]

export WEBOTS_HOME=/mnt/c/Program\ Files/Webots

If installed from sources, source your ROS 2 workspace, if not done already. [待校准@12467]

cd ~/ros2_ws
source install/local_setup.bash

Use the ROS 2 launch command to start demo packages (e.g. webots_ros2_universal_robot). [待校准@12469]

ros2 launch webots_ros2_universal_robot multirobot_launch.py

5 RViz troubleshooting [待校准@12509]

With recent versions of WSL2, RViz should work out of the box. [待校准@12510]

You can check if it works correctly by running any example that uses RViz, for example: [待校准@12511]

sudo apt install ros-humble-slam-toolbox
ros2 launch webots_ros2_tiago robot_launch.py rviz:=true slam:=true

The Tiago robot can be controlled using: [待校准@12512]

ros2 run teleop_twist_keyboard teleop_twist_keyboard

With older WSL versions, RViz2 may not work directly, as no display is available. To use RViz, you can either upgrade WSL or enable X11 forwarding. [待校准@12513]

In a Windows shell: [待校准@12515]

wsl --update

For older versions of WSL, the following steps can be followed: [待校准@12517]

  1. Install VcXsrv. [待校准@12518]

  2. Launch VcXsrv. You can leave most of the parameters default, except the Extra settings page, where you must set Clipboard, Primary Selection and Disable access control and unset Native opengl. [待校准@12519]

  3. You can save the configuration for future launches. [待校准@12520]

  4. Click on Finish, you will see that the X11 server is running in the icon tray. [待校准@12521]

  5. In your WSL environment, export the DISPLAY variable. [待校准@12522]

    export DISPLAY=$(ip route list default | awk '{print }'):0

    You can add this to your .bashrc, so that it is set for every future WSL environment. [待校准@12523]

    echo "export DISPLAY=$(ip route list default | awk '{print }'):0" >> ~/.bashrc
