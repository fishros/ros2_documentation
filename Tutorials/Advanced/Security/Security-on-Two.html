Ensuring security across machines [待校准@12315]

Goal: Make two different machines communicate securely. [待校准@12316]

Tutorial level: Advanced [待校准@11828]

Time: 5 minutes [待校准@12317]

Contents [待校准@12037]

  • Background [待校准@11832]

  • Create the second keystore [待校准@12320]

  • Copy files [待校准@12322]

  • Launch the nodes [待校准@12326]

Background [待校准@11832]

The previous tutorials have used two ROS nodes on the same machine sending all network communications over the localhost interface. Let's extend that scenario to involve multiple machines, since the benefits of authentication and encryption then become more obvious. [待校准@12318]

Suppose that the machine with the keystore created in the previous demo has a hostname Alice, and that we want to also use another machine with hostname Bob for our multi-machine talker/listener demo. We need to move some keys from Alice to Bob to allow SROS 2 to authenticate and encrypt the transmissions. [待校准@12319]

Create the second keystore [待校准@12320]

Begin by creating an empty keystore on Bob; the keystore is actually just an empty directory: [待校准@12321]

ssh Bob
mkdir ~/sros2_demo
exit

ssh Bob
mkdir ~/sros2_demo
exit

ssh Bob
md C:\dev\ros2\sros2_demo
exit

Copy files [待校准@12322]

Next copy the keys and certificates for the talker program from Alice to Bob. Since the keys are just text files, we can use scp to copy them. [待校准@12323]

cd ~/sros2_demo/demo_keystore
scp -r talker USERNAME@Bob:~/sros2_demo/demo_keystore

cd ~/sros2_demo/demo_keystore
scp -r talker USERNAME@Bob:~/sros2_demo/demo_keystore

cd C:\dev\ros2\sros2_demo\demo_keystore
scp -r talker USERNAME@Bob:/dev/ros2/sros2_demo/demo_keystore

警告

Note that in this case the entire keystore is shared across the different machines which may not be the desired behavior, as it may result in a security risk. Please refer to Deployment Guidelines [待校准@12166] for more information in this regard. [待校准@12324]

That will be very quick, since it's just copying some very small text files. Now, we're ready to run a multi-machine talker/listener demo! [待校准@12325]

Launch the nodes [待校准@12326]

Once the environment is set up, run the talker on Bob: [待校准@12327]

ros2 run demo_nodes_cpp talker --ros-args --enclave /talker_listener/talker

and launch the listener on Alice: [待校准@12328]

ros2 run demo_nodes_py listener --ros-args --enclave /talker_listener/listener

Alice will now be receiving encrypted messages from Bob. [待校准@12329]

With two machines successfully communicating using both encryption and authentication, you can use the same procedure to add more machines to your ROS graph. [待校准@12330]
