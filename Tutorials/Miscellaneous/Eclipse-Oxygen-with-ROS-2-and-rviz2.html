Setup [待校准@6576]

This tutorial assumes Eclipse Oxygen, git, and Egit (http://www.eclipse.org/egit/download/) are already installed. [待校准@6577]

Throughout the tutorial we name the eclipse workspace the same name as the ros2 package, but this is not required. [待校准@6578]

HINT: We use nested projects and one Eclipse Workspace for each ROS-2 package. [待校准@6579]

Create a C++ Project. [待校准@6580]

Choose the ROS 2 package name as the Project Name. Choose a Makefile Project and Other Toolchain. [待校准@6581]

Click on Finish [待校准@6582]

Our project should be shown in the "Project Explorer". [待校准@6583]

Inside our Project create a folder called "src". [待校准@6584]

Import a git repository. [待校准@6585]

Put in the repository URL. [待校准@6586]

IMPORTANT: Use the source folder of the project we created before as the destination folder. [待校准@6587]

HINT: If you ran into problems choosing the destination folder path, the Eclipse Dialog needs a name in the name field. [待校准@6588]

Import using the new project wizard. [待校准@6589]

Create a General->Project. [待校准@6590]

Use the git repository name as the project name. IMPORTANT: Use the folder we cloned the git repository in as the "Location". [待校准@6591]

The git project and the new project should be visible in the Project Explorer view. The same files are listed multiple times, but only one project is linked with Egit. [待校准@6592]

Repeat this procedure again. Import git repository pluginlib. [待校准@6593]

IMPORTANT: Use a folder inside the source folder as "Destination->Directory". [待校准@6594]

IMPORTANT: Use the folder we cloned the git repository in as the location for the new project. [待校准@6595]

Run the same procedure with the tinyxml2_vendor git repository. [待校准@6596]

IMPORTANT: Again use a folder inside the source folder. [待校准@6597]

IMPORTANT: Use the location of the folder we cloned as the new project folder. [待校准@6598]

Now all four Projects should be visible in the Project Explorer view. [待校准@6599]

Clicking in the top right cornder for the Project Explorer view allows us to change the Project Presentation to Hierarchical view. Now it looks like a ROS-2 project as it is on the hard drive. But this view loses the linkage to Egit, so use the Flat Project Presentation. The Egit linkage is good if you want to see e.g. which author wrote which code-line, etc. [待校准@6600]

Go to "C/C++ build"-section and put "ament" into "Build command". [待校准@6601]

Go to "Behavior" tab and unselect "clean" and put "build" into Build textbox. [待校准@6602]

Before "Build project" will work, we need to close Eclipse. Open a shell and source the ROS-2 setup.bash file, then cd into the directory of the eclipse project (here: /home/ubu/rviz2_ws/rviz2_ws) and start Eclipse from inside this directory. [待校准@6603]

Now code completion, egit annotations, eclipse C/C++ Tools, etc. should all work. [待校准@6604]

Eclipse-indexer [待校准@6605]

Opening the main.cpp of rviz2 may show a lot of "unresolved inclusion" warnings. To fix this, go to Project->Properties->C++ General->Path and Symbols. Click on the "References" tab and select "ros2_ws". [待校准@6606]

Go to C/C++-General->Path-and-Symbols, click on the "Source locations" tab and click on "Link folder". Choose the location of qt5 includes. [待校准@6607]

The next image should be shown. It is a good idea to add excludes to the source locations, so that some directories (like "Build" and "Install") don't get indexed. [待校准@6608]

Go to C++General->Preprocessor includes, select "CDT GCC Built in compiler settings [Shared]" and enter in the "command to get compiler specs" text box the following: [待校准@6609]

-std=c++14

Go to "C/C++-General->Indexer" and select the following in the image. E.g "index unused headers as c files" to resolve e.g. QApplication, because the QApplication headers content is only "#include "qapplication.h". [待校准@6610]

After running the indexer (which happens later, so you will see this also later), you can see what it added [待校准@6611]

After that right-click on the rviz2 project and select "Indexer->Rebuild", which will start rebuilding the index (there is an icon in the lower right showing progress). Once the index is finished rebuilding, it should be able to resolve all includes. [待校准@6612]

-DCMAKE_BUILD_TYPE=Debug
