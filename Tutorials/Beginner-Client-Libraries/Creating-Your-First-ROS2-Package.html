Background [待校准@3072]

1 What is a ROS 2 package? [待校准@4498]

A package is an organizational unit for your ROS 2 code. If you want to be able to install your code or share it with others, then you'll need it organized in a package. With packages, you can release your ROS 2 work and allow others to build and use it easily. [待校准@4499]

Package creation in ROS 2 uses ament as its build system and colcon as its build tool. You can create a package using either CMake or Python, which are officially supported, though other build types do exist. [待校准@4500]

2 What makes up a ROS 2 package? [待校准@4501]

ROS 2 Python and CMake packages each have their own minimum required contents: [待校准@4502]

  • CMakeLists.txt file that describes how to build the code within the package [待校准@4504]

  • include/<package_name> directory containing the public headers for the package [待校准@4505]

  • package.xml file containing meta information about the package [待校准@4506]

  • src directory containing the source code for the package [待校准@4507]

  • package.xml file containing meta information about the package [待校准@4506]

  • resource/<package_name> marker file for the package [待校准@4508]

  • setup.cfg is required when a package has executables, so ros2 run can find them [待校准@4509]

  • setup.py containing instructions for how to install the package [待校准@4510]

  • <package_name> - a directory with the same name as your package, used by ROS 2 tools to find your package, contains __init__.py [待校准@4511]

The simplest possible package may have a file structure that looks like: [待校准@4512]

my_package/
     CMakeLists.txt
     include/my_package/
     package.xml
     src/

my_package/
      package.xml
      resource/my_package
      setup.cfg
      setup.py
      my_package/

3 Packages in a workspace [待校准@4513]

A single workspace can contain as many packages as you want, each in their own folder. You can also have packages of different build types in one workspace (CMake, Python, etc.). You cannot have nested packages. [待校准@4514]

Best practice is to have a src folder within your workspace, and to create your packages in there. This keeps the top level of the workspace “clean”. [待校准@4515]

A trivial workspace might look like: [待校准@4516]

workspace_folder/
    src/
      cpp_package_1/
          CMakeLists.txt
          include/cpp_package_1/
          package.xml
          src/

      py_package_1/
          package.xml
          resource/py_package_1
          setup.cfg
          setup.py
          py_package_1/
      ...
      cpp_package_n/
          CMakeLists.txt
          include/cpp_package_n/
          package.xml
          src/

Prerequisites [待校准@3124]

You should have a ROS 2 workspace after following the instructions in the previous tutorial. You will create your package in this workspace. [待校准@4517]

Tasks [待校准@3282]

1 Create a package [待校准@3283]

First, source your ROS 2 installation. [待校准@4518]

Let's use the workspace you created in the previous tutorial, ros2_ws, for your new package. [待校准@4519]

Make sure you are in the src folder before running the package creation command. [待校准@4520]

cd ~/ros2_ws/src

cd ~/ros2_ws/src

cd \ros2_ws\src

The command syntax for creating a new package in ROS 2 is: [待校准@4521]

ros2 pkg create --build-type ament_cmake <package_name>

ros2 pkg create --build-type ament_python <package_name>

For this tutorial, you will use the optional argument --node-name which creates a simple Hello World type executable in the package. [待校准@4522]

Enter the following command in your terminal: [待校准@4523]

ros2 pkg create --build-type ament_cmake --node-name my_node my_package

ros2 pkg create --build-type ament_python --node-name my_node my_package

You will now have a new folder within your workspace's src directory called my_package. [待校准@4524]

2 Build a package [待校准@4527]

Putting packages in a workspace is especially valuable because you can build many packages at once by running colcon build in the workspace root. Otherwise, you would have to build each package individually. [待校准@4528]

Return to the root of your workspace: [待校准@4529]

cd ~/ros2_ws

cd ~/ros2_ws

cd \ros2_ws

Now you can build your packages: [待校准@4530]

colcon build

colcon build

colcon build --merge-install

Windows doesn't allow long paths, so merge-install will combine all the paths into the install directory. [待校准@4397]

Recall from the last tutorial that you also have the ros_tutorials packages in your ros2_ws. You might have noticed that running colcon build also built the turtlesim package. That's fine when you only have a few packages in your workspace, but when there are many packages, colcon build can take a long time. [待校准@4531]

To build only the my_package package next time, you can run: [待校准@4532]

colcon build --packages-select my_package

3 Source the setup file [待校准@4533]

To use your new package and executable, first open a new terminal and source your main ROS 2 installation. [待校准@4534]

Then, from inside the ros2_ws directory, run the following command to source your workspace: [待校准@4535]

source install/local_setup.bash

. install/local_setup.bash

call install/local_setup.bat

Now that your workspace has been added to your path, you will be able to use your new package's executables. [待校准@4536]

4 Use the package [待校准@4537]

To run the executable you created using the --node-name argument during package creation, enter the command: [待校准@4538]

ros2 run my_package my_node

Which will return a message to your terminal: [待校准@4539]

hello world my_package package

Hi from my_package.

5 Examine package contents [待校准@4540]

Inside ros2_ws/src/my_package, you will see the files and folders that ros2 pkg create automatically generated: [待校准@4541]

CMakeLists.txt  include  package.xml  src

my_node.cpp is inside the src directory. This is where all your custom C++ nodes will go in the future. [待校准@4542]

my_package  package.xml  resource  setup.cfg  setup.py  test

my_node.py is inside the my_package directory. This is where all your custom Python nodes will go in the future. [待校准@4543]

6 Customize package.xml [待校准@4544]

You may have noticed in the return message after creating your package that the fields description and license contain TODO notes. That's because the package description and license declaration are not automatically set, but are required if you ever want to release your package. The maintainer field may also need to be filled in. [待校准@4545]

From ros2_ws/src/my_package, open package.xml using your preferred text editor: [待校准@4546]

<?xml version="1.0"?>
<?xml-model
   href="http://download.ros.org/schema/package_format3.xsd"
   schematypens="http://www.w3.org/2001/XMLSchema"?>
<package format="3">
 <name>my_package</name>
 <version>0.0.0</version>
 <description>TODO: Package description</description>
 <maintainer email="user@todo.todo">user</maintainer>
 <license>TODO: License declaration</license>

 <buildtool_depend>ament_cmake</buildtool_depend>
