Implementing custom interfaces [待校准@4703]

Goal: Learn more ways to implement custom interfaces in ROS 2. [待校准@4704]

Tutorial level: Beginner [待校准@3951]

Time: 15 minutes [待校准@3507]

Background [待校准@3072]

In a previous tutorial, you learned how to create custom msg and srv interfaces. [待校准@4705]

While best practice is to declare interfaces in dedicated interface packages, sometimes it can be convenient to declare, create and use an interface all in one package. [待校准@4706]

Recall that interfaces can currently only be defined in CMake packages. It is possible, however, to have Python libraries and nodes in CMake packages (using ament_cmake_python), so you could define interfaces and Python nodes together in one package. We'll use a CMake package and C++ nodes here for the sake of simplicity. [待校准@4707]

This tutorial will focus on the msg interface type, but the steps here are applicable to all interface types. [待校准@4708]

Prerequisites [待校准@3124]

We assume you've reviewed the basics in the Creating custom msg and srv files [待校准@4559] tutorial before working through this one. [待校准@4709]

You should have ROS 2 installed, a workspace, and an understanding of creating packages. [待校准@4710]

As always, don't forget to source ROS 2 in every new terminal you open. [待校准@4711]

Tasks [待校准@3282]

1 Create a package [待校准@3283]

In your workspace src directory, create a package more_interfaces and make a directory within it for msg files: [待校准@4712]

ros2 pkg create --build-type ament_cmake more_interfaces
mkdir more_interfaces/msg

2 Create a msg file [待校准@4713]

Inside more_interfaces/msg, create a new file AddressBook.msg, and paste the following code to create a message meant to carry information about an individual: [待校准@4714]

uint8 PHONE_TYPE_HOME=0
uint8 PHONE_TYPE_WORK=1
uint8 PHONE_TYPE_MOBILE=2

string first_name
string last_name
string phone_number
uint8 phone_type

This message is composed of these fields: [待校准@4715]

  • first_name: of type string [待校准@4716]

  • last_name: of type string [待校准@4717]

  • phone_number: of type string [待校准@4718]

  • phone_type: of type uint8, with several named constant values defined [待校准@4719]

Note that it's possible to set default values for fields within a message definition. See About ROS 2 interfaces [待校准@7285] for more ways you can customize interfaces. [待校准@4720]

Next, we need to make sure that the msg file is turned into source code for C++, Python, and other languages. [待校准@4721]

2.1 Build a msg file [待校准@4722]

Open package.xml and add the following lines: [待校准@4723]

<buildtool_depend>rosidl_default_generators</buildtool_depend>

<exec_depend>rosidl_default_runtime</exec_depend>

<member_of_group>rosidl_interface_packages</member_of_group>

Note that at build time, we need rosidl_default_generators, while at runtime, we only need rosidl_default_runtime. [待校准@4724]

Open CMakeLists.txt and add the following lines: [待校准@4725]

Find the package that generates message code from msg/srv files: [待校准@4726]

find_package(rosidl_default_generators REQUIRED)

Declare the list of messages you want to generate: [待校准@4727]

set(msg_files
  "msg/AddressBook.msg"
)

By adding the .msg files manually, we make sure that CMake knows when it has to reconfigure the project after you add other .msg files. [待校准@4728]

Generate the messages: [待校准@4729]

rosidl_generate_interfaces(${PROJECT_NAME}
  ${msg_files}
)

Also make sure you export the message runtime dependency: [待校准@4730]

ament_export_dependencies(rosidl_default_runtime)

Now you're ready to generate source files from your msg definition. We'll skip the compile step for now as we'll do it all together below in step 4. [待校准@4731]

2.2 (Extra) Set multiple interfaces [待校准@4732]

注解

You can use set in CMakeLists.txt to neatly list all of your interfaces: [待校准@4733]

set(msg_files
  "msg/Message1.msg"
  "msg/Message2.msg"
  # etc
  )

set(srv_files
  "srv/Service1.srv"
  "srv/Service2.srv"
   # etc
  )

And generate all lists at once like so: [待校准@4734]

rosidl_generate_interfaces(${PROJECT_NAME}
  ${msg_files}
  ${srv_files}
)

3 Use an interface from the same package [待校准@4735]

Now we can start writing code that uses this message. [待校准@4736]

In more_interfaces/src create a file called publish_address_book.cpp and paste the following code: [待校准@4737]

#include <chrono>
#include <memory>

#include "rclcpp/rclcpp.hpp"
#include "more_interfaces/msg/address_book.hpp"

using namespace std::chrono_literals;

class AddressBookPublisher : public rclcpp::Node
{
public:
  AddressBookPublisher()
  : Node("address_book_publisher")
  {
    address_book_publisher_ =
      this->create_publisher<more_interfaces::msg::AddressBook>("address_book", 10);

    auto publish_msg = [this]() -> void {
        auto message = more_interfaces::msg::AddressBook();

        message.first_name = "John";
        message.last_name = "Doe";
        message.phone_number = "1234567890";
        message.phone_type = message.PHONE_TYPE_MOBILE;

        std::cout << "Publishing Contact\nFirst:" << message.first_name <<
          "  Last:" << message.last_name << std::endl;

        this->address_book_publisher_->publish(message);
      };
    timer_ = this->create_wall_timer(1s, publish_msg);
  }

private:
  rclcpp::Publisher<more_interfaces::msg::AddressBook>::SharedPtr address_book_publisher_;
  rclcpp::TimerBase::SharedPtr timer_;
};


int main(int argc, char * argv[])
{
  rclcpp::init(argc, argv);
  rclcpp::spin(std::make_shared<AddressBookPublisher>());
  rclcpp::shutdown();

  return 0;
}

3.1 The code explained [待校准@4738]

Include the header of our newly created AddressBook.msg. [待校准@4739]

#include "more_interfaces/msg/address_book.hpp"

Create a node and an AddressBook publisher. [待校准@4740]

using namespace std::chrono_literals;

class AddressBookPublisher : public rclcpp::Node
{
public:
  AddressBookPublisher()
  : Node("address_book_publisher")
  {
    address_book_publisher_ =
      this->create_publisher<more_interfaces::msg::AddressBook>("address_book");

Create a callback to publish the messages periodically. [待校准@4741]

auto publish_msg = [this]() -> void {

Create an AddressBook message instance that we will later publish. [待校准@4742]

auto message = more_interfaces::msg::AddressBook();

Populate AddressBook fields. [待校准@4743]

message.first_name = "John";
message.last_name = "Doe";
message.phone_number = "1234567890";
message.phone_type = message.PHONE_TYPE_MOBILE;

Finally send the message periodically. [待校准@4744]

std::cout << "Publishing Contact\nFirst:" << message.first_name <<
  "  Last:" << message.last_name << std::endl;

this->address_book_publisher_->publish(message);

Create a 1 second timer to call our publish_msg function every second. [待校准@4745]

timer_ = this->create_wall_timer(1s, publish_msg);

3.2 Build the publisher [待校准@4746]

We need to create a new target for this node in the CMakeLists.txt: [待校准@4747]

find_package(rclcpp REQUIRED)

add_executable(publish_address_book src/publish_address_book.cpp)
ament_target_dependencies(publish_address_book rclcpp)

install(TARGETS
    publish_address_book
  DESTINATION lib/${PROJECT_NAME})

3.3 Link against the interface [待校准@4748]

In order to use the messages generated in the same package we need to use the following CMake code: [待校准@4749]

rosidl_get_typesupport_target(cpp_typesupport_target
  ${PROJECT_NAME} rosidl_typesupport_cpp)

target_link_libraries(publish_address_book "${cpp_typesupport_target}")

This finds the relevant generated C++ code from AddressBook.msg and allows your target to link against it. [待校准@4750]

You may have noticed that this step was not necessary when the interfaces being used were from a different package that was built independently. This CMake code is only required when you want to use interfaces in the same package as the one in which they are defined. [待校准@4751]

4 Try it out [待校准@4752]

Return to the root of the workspace to build the package: [待校准@4753]

cd ~/ros2_ws
colcon build --packages-up-to more_interfaces

cd ~/ros2_ws
colcon build --packages-up-to more_interfaces

cd /ros2_ws
colcon build --merge-install --packages-up-to more_interfaces

Then source the workspace and run the publisher: [待校准@4754]

source install/local_setup.bash
ros2 run more_interfaces publish_address_book

. install/local_setup.bash
ros2 run more_interfaces publish_address_book

call install/local_setup.bat
ros2 run more_interfaces publish_address_book

5 (Extra) Use an existing interface definition [待校准@4759]

注解

You can use an existing interface definition in a new interface definition. For example, let's say there is a message named Contact.msg that belongs to an existing ROS 2 package named rosidl_tutorials_msgs. Assume that its definition is identical to our custom-made AddressBook.msg interface from earlier. [待校准@4760]

In that case you could have defined AddressBook.msg (an interface in the package with your nodes) as type Contact (an interface in a separate package). You could even define AddressBook.msg as an array of type Contact, like so: [待校准@4761]

rosidl_tutorials_msgs/Contact[] address_book

To generate this message you would need to declare a dependency on Contact.msg's package, rosidl_tutorials_msgs, in package.xml: [待校准@4762]

<build_depend>rosidl_tutorials_msgs</build_depend>

<exec_depend>rosidl_tutorials_msgs</exec_depend>

And in CMakeLists.txt: [待校准@4763]

find_package(rosidl_tutorials_msgs REQUIRED)

rosidl_generate_interfaces(${PROJECT_NAME}
  ${msg_files}
  DEPENDENCIES rosidl_tutorials_msgs
)

You would also need to include the header of Contact.msg in you publisher node in order to be able to add contacts to your address_book. [待校准@4764]

#include "rosidl_tutorials_msgs/msg/contact.hpp"

You could change the callback to something like this: [待校准@4765]

auto publish_msg = [this]() -> void {
   auto msg = std::make_shared<more_interfaces::msg::AddressBook>();
   {
     rosidl_tutorials_msgs::msg::Contact contact;
     contact.first_name = "John";
     contact.last_name = "Doe";
     contact.phone_number = "1234567890";
     contact.phone_type = message.PHONE_TYPE_MOBILE;
     msg->address_book.push_back(contact);
   }
   {
     rosidl_tutorials_msgs::msg::Contact contact;
     contact.first_name = "Jane";
     contact.last_name = "Doe";
     contact.phone_number = "4254242424";
     contact.phone_type = message.PHONE_TYPE_HOME;
     msg->address_book.push_back(contact);
   }

   std::cout << "Publishing address book:" << std::endl;
   for (auto contact : msg->address_book) {
     std::cout << "First:" << contact.first_name << "  Last:" << contact.last_name <<
       std::endl;
   }

   address_book_publisher_->publish(*msg);
 };

Building and running these changes would show the msg defined as expected, as well as the array of msgs defined above. [待校准@4766]
