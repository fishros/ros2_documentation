Traveling in time (C++) [待校准@5924]

Goal: Learn about advanced time travel features of tf2. [待校准@5925]

Time: 10 minutes [待校准@3663]

Background [待校准@3072]

In the previous tutorial, we discussed the basics of tf2 and time. This tutorial will take us one step further and expose a powerful tf2 trick: the time travel. In short, one of the key features of tf2 library is that it is able to transform data in time as well as in space. [待校准@5926]

This tf2 time travel feature can be useful for various tasks, like monitoring the pose of the robot for a long period of time or building a follower robot that will follow the "steps" of the leader. We will use that time travel feature to look up transforms back in time and program turtle2 to follow 5 seconds behind carrot1. [待校准@5927]

Time travel [待校准@5928]

First, let's go back to where we ended in the previous tutorial Using time. Go to your learning_tf2_cpp package. [待校准@5929]

Now, instead of making the second turtle go to where the carrot is now, we will make the second turtle go to where the first carrot was 5 seconds ago. Edit the lookupTransform() call in turtle_tf2_listener.cpp file to [待校准@5930]

rclcpp::Time when = this->get_clock()->now() - rclcpp::Duration(5, 0);
t = tf_buffer_->lookupTransform(
    toFrameRel,
    fromFrameRel,
    when,
    50ms);

Now if you run this, during the first 5 seconds, the second turtle would not know where to go because we do not yet have a 5-second history of poses of the carrot. But what happens after these 5 seconds? Let's just give it a try: [待校准@5931]

ros2 launch learning_tf2_cpp turtle_tf2_fixed_frame_demo.launch.py

You should now notice that your turtle is driving around uncontrollably like in this screenshot. Let's try to understand reason behind that behavior. [待校准@5932]

  1. In our code we asked tf2 the following question: "What was the pose of carrot1 5 seconds ago, relative to turtle2 5 seconds ago?". This means we are controlling the second turtle based on where it was 5 seconds ago as well as where the first carrot was 5 seconds ago. [待校准@5933]

  2. However, what we really want to ask is: "What was the pose of carrot1 5 seconds ago, relative to the current position of the turtle2?". [待校准@5934]

Advanced API for lookupTransform() [待校准@5935]

To ask the tf2 that particular question, we will use an advanced API that gives us the power to say explicitly when to acquire the specified transformations. This is done by calling the lookupTransform() method with additional parameters. Your code now would look like this: [待校准@5936]

rclcpp::Time now = this->get_clock()->now();
rclcpp::Time when = now - rclcpp::Duration(5, 0);
t = tf_buffer_->lookupTransform(
    toFrameRel,
    now,
    fromFrameRel,
    when,
    "world",
    50ms);

The advanced API for lookupTransform() takes six arguments: [待校准@5937]

  1. Target frame [待校准@5938]

  2. The time to transform to [待校准@5939]

  3. Source frame [待校准@5940]

  4. The time at which source frame will be evaluated [待校准@5941]

  5. Frame that does not change over time, in this case the world frame [待校准@5942]

  6. Time to wait for the target frame to become available [待校准@5943]

To sum up, tf2 does the following in the background. In the past, it computes the transform from the carrot1 to the world. In the world frame, tf2 time travels from the past to now. And at the current time, tf2 computes the transform from the world to the turtle2. [待校准@5944]

Checking the results [待校准@5945]

Let's run the simulation again, this time with the advanced time-travel API: [待校准@5946]

ros2 launch learning_tf2_cpp turtle_tf2_fixed_frame_demo.launch.py

And yes, the second turtle is directed to where the first carrot was 5 seconds ago! [待校准@5947]

Summary [待校准@3353]

In this tutorial, you have seen one of the advanced features of tf2. You learned that tf2 can transform data in time and learned how to do that with turtlesim example. tf2 allowed you to go back in time and make frame transformations between old and current poses of turtles by using the advanced lookupTransform() API. [待校准@5948]
