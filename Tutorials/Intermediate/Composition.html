Tutorial level: Intermediate [待校准@14046]

Time: 20 minutes [待校准@11829]

Background [待校准@11832]

See the conceptual article. [待校准@14047]

Run the demos [待校准@14048]

The demos use executables from rclcpp_components, ros2component, and composition packages, and can be run with the following commands. [待校准@14049]

Discover available components [待校准@14050]

To see what components are registered and available in the workspace, execute the following in a shell: [待校准@14051]

ros2 component types

The terminal will return the list of all available components: [待校准@14052]

(... components of other packages here)
composition
  composition::Talker
  composition::Listener
  composition::NodeLikeListener
  composition::Server
  composition::Client
(... components of other packages here)

Run-time composition using ROS services with a publisher and subscriber [待校准@14053]

In the first shell, start the component container: [待校准@14054]

ros2 run rclcpp_components component_container

Open the second shell and verify that the container is running via ros2 command line tools: [待校准@14055]

ros2 component list

You should see a name of the component: [待校准@14056]

/ComponentManager

In the second shell load the talker component (see talker source code): [待校准@14057]

ros2 component load /ComponentManager composition composition::Talker

The command will return the unique ID of the loaded component as well as the node name: [待校准@14058]

Loaded component 1 into '/ComponentManager' container node as '/talker'

Now the first shell should show a message that the component was loaded as well as repeated message for publishing a message. [待校准@14059]

Run another command in the second shell to load the listener component (see listener source code): [待校准@14060]

ros2 component load /ComponentManager composition composition::Listener

Terminal will return: [待校准@14061]

Loaded component 2 into '/ComponentManager' container node as '/listener'

The ros2 command line utility can now be used to inspect the state of the container: [待校准@14062]

ros2 component list

You will see the following result: [待校准@14063]

/ComponentManager
   1  /talker
   2  /listener

Now the first shell should show repeated output for each received message. [待校准@14064]

Run-time composition using ROS services with a server and client [待校准@14065]

The example with a server and a client is very similar. [待校准@14066]

In the first shell: [待校准@14067]

ros2 run rclcpp_components component_container

In the second shell (see server and client source code): [待校准@14068]

ros2 component load /ComponentManager composition composition::Server
ros2 component load /ComponentManager composition composition::Client

In this case the client sends a request to the server, the server processes the request and replies with a response, and the client prints the received response. [待校准@14069]

Compile-time composition using ROS services [待校准@14070]

This demos shows that the same shared libraries can be reused to compile a single executable running multiple components. The executable contains all four components from above: talker and listener as well as server and client. [待校准@14071]

In the shell call (see source code): [待校准@14072]

ros2 run composition manual_composition

This should show repeated messages from both pairs, the talker and the listener as well as the server and the client. [待校准@14073]

注解

Manually-composed components will not be reflected in the ros2 component list command line tool output. [待校准@14074]

Run-time composition using dlopen [待校准@14075]

This demo presents an alternative to run-time composition by creating a generic container process and explicitly passing the libraries to load without using ROS interfaces. The process will open each library and create one instance of each "rclcpp::Node" class in the library source code). [待校准@14076]

ros2 run composition dlopen_composition `ros2 pkg prefix composition`/lib/libtalker_component.so `ros2 pkg prefix composition`/lib/liblistener_component.so

ros2 run composition dlopen_composition `ros2 pkg prefix composition`/lib/libtalker_component.dylib `ros2 pkg prefix composition`/lib/liblistener_component.dylib

> ros2 pkg prefix composition

to get the path to where composition is installed. Then call [待校准@14077]

> ros2 run composition dlopen_composition <path_to_composition_install>\bin\talker_component.dll <path_to_composition_install>\bin\listener_component.dll

Now the shell should show repeated output for each sent and received message. [待校准@14078]

注解

dlopen-composed components will not be reflected in the ros2 component list command line tool output. [待校准@14079]

Composition using launch actions [待校准@14080]

While the command line tools are useful for debugging and diagnosing component configurations, it is frequently more convenient to start a set of components at the same time. To automate this action, we can use a launch file: [待校准@14081]

ros2 launch composition composition_demo.launch.py

Advanced Topics [待校准@14082]

Now that we have seen the basic operation of components, we can discuss a few more advanced topics. [待校准@14083]

Unloading components [待校准@14084]

In the first shell, start the component container: [待校准@14054]

ros2 run rclcpp_components component_container

Verify that the container is running via ros2 command line tools: [待校准@14085]

ros2 component list

You should see a name of the component: [待校准@14056]

/ComponentManager

In the second shell load both the talker and listener as we have before: [待校准@14086]

ros2 component load /ComponentManager composition composition::Talker
ros2 component load /ComponentManager composition composition::Listener

Use the unique ID to unload the node from the component container. [待校准@14087]

ros2 component unload /ComponentManager 1 2

The terminal should return: [待校准@14088]

Unloaded component 1 from '/ComponentManager' container
Unloaded component 2 from '/ComponentManager' container

In the first shell, verify that the repeated messages from talker and listener have stopped. [待校准@14089]

Remapping container name and namespace [待校准@14090]

The component manager name and namespace can be remapped via standard command line arguments: [待校准@14091]

ros2 run rclcpp_components component_container --ros-args -r __node:=MyContainer -r __ns:=/ns

In a second shell, components can be loaded by using the updated container name: [待校准@14092]

ros2 component load /ns/MyContainer composition composition::Listener

注解

Namespace remappings of the container do not affect loaded components. [待校准@14093]

Remap component names and namespaces [待校准@14094]

Component names and namespaces may be adjusted via arguments to the load command. [待校准@14095]

In the first shell, start the component container: [待校准@14054]

ros2 run rclcpp_components component_container

Some examples of how to remap names and namespaces. [待校准@14096]

Remap node name: [待校准@14097]

ros2 component load /ComponentManager composition composition::Talker --node-name talker2

Remap namespace: [待校准@14098]

ros2 component load /ComponentManager composition composition::Talker --node-namespace /ns

Remap both: [待校准@14099]

ros2 component load /ComponentManager composition composition::Talker --node-name talker3 --node-namespace /ns2

Now use ros2 command line utility: [待校准@14100]

ros2 component list

In the console you should see corresponding entries: [待校准@14101]

/ComponentManager
   1  /talker2
   2  /ns/talker
   3  /ns2/talker3

注解

Namespace remappings of the container do not affect loaded components. [待校准@14093]

Passing parameter values into components [待校准@14102]

The ros2 component load command-line supports passing arbitrary parameters to the node as it is constructed. This functionality can be used as follows: [待校准@14103]

ros2 component load /ComponentManager image_tools image_tools::Cam2Image -p burger_mode:=true

Passing additional arguments into components [待校准@14104]

The ros2 component load command-line supports passing particular options to the component manager for use when constructing the node. As of now, the only command-line option that is supported is to instantiate a node using intra-process communication. This functionality can be used as follows: [待校准@14105]

ros2 component load /ComponentManager composition composition::Talker -e use_intra_process_comms:=true

Composable nodes as shared libraries [待校准@14106]

If you want to export a composable node as a shared library from a package and use that node in another package that does link-time composition, add code to the CMake file which imports the actual targets in downstream packages. [待校准@14107]

Then install the generated file and export the generated file. [待校准@14108]

A practical example can be seen here: ROS Discourse - Ament best practice for sharing libraries [待校准@14109]

Composing Non-Node Derived Components [待校准@14110]

In ROS 2, components allow for more efficient use of system resources and provide a powerful feature that enables you to create reusable functionality that is not tied to a specific node. [待校准@14111]

One advantage of using components is that they allow you to create non-node derived functionality as standalone executables or shared libraries that can be loaded into the ROS system as needed. [待校准@14112]

To create a component that is not derived from a node, follow these guidelines: [待校准@14113]

  1. Implement a constructor that takes const rclcpp::NodeOptions& as its argument. [待校准@14114]

  2. Implement the get_node_base_interface() method, which should return a NodeBaseInterface::SharedPtr. You can use the get_node_base_interface() method of a node that you create in your constructor to provide this interface. [待校准@14115]

Here's an example of a component that is not derived from a node, which listens to a ROS topic: node_like_listener_component. [待校准@14116]

For more information on this topic, you can refer to this discussion. [待校准@14117]
